Getcell
   

This method reads a cell contents from a grid.

Format

            Getcell(line,column)

Parameters


·     line - grid line, numeric value, integer. The first line is 1, etc. Special value is CURRENT_LINE. The current line is the selected one in display-only grid, or a line containing the cursor in data-entry grid.
·     column - grid column, numeric value, integer. The first column is 1.
                        Instead of numeric value, column identifier can be used. Column identifier is name of the column field.

Return Value

            Value of the cell.

Applied To

            Grid.


Notes

            If you want to read cell value as an alphanumeric one, just enter the command in the form

                        $alpha_variable = Table_n.Getcell(line,column)

            In that case, output alpha variable contains the value you see in the grid, like 1.000,00 or 01/01/2015 or 10:15:00, etc.

            But, if you know that a particular column contains numeric value, and you want to read this value in numeric form suitable for calculations, the command is

                        $num_variable = Table_n.Getcell(line,column)

            If line or column is invalid (< 1 or > number of lines / columns) or type incompatibility occurs (for example, try to move date cell into a numeric variable), output variable will be initialized.


            If we try to read a CHECKBOX column, result is 0 (not checked) or 1 (checked) despite of check box definition (Destination=field/checked:"X"/unchecked:"Y").


Examples

  
      $var  = Table_1.Getcell(current_line,1)   !!column number
      acc_1 = Table_1.Getcell(current_line,2)
      acc_1 = Table_1.Getcell(counter_1,counter_2)
      acc_1 = Table_1.Getcell(counter_1,colID) !!column identifier
      acc_1 = Table_1.Getcell(current_line,colID)

! Variable object name
      if $flag = 1
            $name = 'Table_1'
      else
            $name = 'Table_2'
      end_if
      $name.Getcell($line,$col)
