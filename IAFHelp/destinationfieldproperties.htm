Destination_field properties
   

Object name

When the new destination_field is created, generator automatically assigns the name Destinationfield_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Field name (information only, cannot be changed)

Name of the destination field. In IAF, data entered via keyboard must be recorded somewhere: into a table field, a local field, a counter, an accumulator. This place we call destination field.

Member of group (information only, cannot be changed)

If the object is a member of a group, the name of this group is displayed. An object can be the member of one group only.

Visible

Determines whether the object is visible when the subform is first displayed or initialized.

Enabled

Determines whether the object is disabled when the subform is first displayed or initialized. When the object is disabled, mouse and keyboard input are ignored.


Required

Specifies whether the value is required in the field. This means the following:
            - an alphanumeric field cannot be blank,
            - a numeric field must be different from zero,
            - a date field must be different from zero_date,
            - a time field must be different from zero_time.

Unique

Applies to the destination field in a data-entry grid.
No two rows in the grid are allowed to have the same values for that column.

Blank when zero

Displays spaces when the value of the field is zero.
Applies to a numeric, date or time field.

Multi-line (alpha fields only)

This option can be used if the alpha field is very long. Number of characters which can be entered is equal to the length of the field. If the field is LONG VARCHAR, field length is not limited.
During the program execution, new line can be opened by pressing Ctrl/Enter, Enter and TAB are normal terminators, and Shift/Tab returns control to the previous field (object).

Vertical scroll

Applies to multi-line fields only. If checked, vertical scroll bar is visible.

Store formatting characters

Applies to alpha fields with edit mask containing formatting characters. If checked, formatting characters will be stored to the output field.

Magnifier on the right

The magnifier appears on the right side of the field. Default is left side.

Autoincrement

Applies to the destination numeric field without decimals in a data-entry grid. Only the field in the first column have this attribute.


Use Enter as a terminator (alpha fields only, multiline)

By default, the Enter key is a terminator in multiline field. If you want to consider the Enter key as an ordinary character, clear this check box. In that case, the Enter key opens new line in the field, and terminator is TAB.

Ignore edit mask (alpha field only)

If 'multi-line' attribute has not been set, data entry is formatted (see formatted_dataentry). Since the maximum length of the edit-mask is 127 characters, by standard data entry (single line) it is possible to enter maximum 127 characters. To overcome those limits, 'Ignore edit mask' attribute should be set. Then the edit-mask is ignored, and the number of characters which can entered is equal to the length of the field. This option is especially appropriate when dealing with the long field, and there is no sufficient place for the large rectangle on the screen.

Background style

Background style should be set to normal.

If background style is transparent, then the background color is taken from subform and during the program execution strange effects appear.

Background color

This attribute is applied only if the background style is normal. Double click brings up a color definition dialog.

Special effects

Values:

Flat
Raised
Sunken
Etched
Bump
Shadowed

Border style

Values:

Transparent      border is not seen,
Solid                 the border is standard line,
Dots                 the border is dotted line,
Dashes             the border is dashed line.

Border color

Is applied only if border style is not transparent.  Double click brings up the color definition dialog.

Border width

Is applied only if border style is not transparent. Possibly values are 0, 1, 2,... pixels.

Text color

Color of text. Double click brings up the color definition dialog.

Font name

Choose one from offered font names.

Font size

Specifies font size of the text.

Bold

Specify that font weight is 700. Normal weight is 400.

Italic

Specify whether text is italic.

Underline

Specify whether text is underlined.

Horizontal text alignment

Values:

left
center
right

Numeric fields are usually right aligned; alpha and date ones are left aligned.

Sequence number

In IAF, the sequence number is very important attribute because it defines the order of the processing within a subform. Only active objects can have the sequence: destination_field, source_field, combobox, checkbox, radiobutton, table (grid), data_selector, button, bitmap_button.

Destination_field must have nonzero sequence number.

Run_time processing:
After subform is displayed, object with the lowest sequence number gets the focus. When the field is entered, object with next sequence gets the focus, etc. After the last object in the subform is processed, there are two possibilities:

1/ if 'Use Iaf program sequence' attribute has been set (see program_properties), subform calculations are activated and, on the basis of the attribute 'next subform', the program execution continues.

2/ if 'Use Iaf program sequence' attribute has not been set, there is no focus. In that case, there must be one button with the sequence 0 on the subform, and in his click actions needs
to exist FINISH_WINDOW instruction.

Tooltip text

Tooltip text is a text that will be displayed when the cursor hovers above the field. See Tooltip text definition.
