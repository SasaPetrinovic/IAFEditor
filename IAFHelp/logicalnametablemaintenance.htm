Logical Name Table Maintenance
   

Logical names table establishes connection between logical name and physical path of a file.
To avoid absolute file references, like c:\data\file1.txt, we use logical names like datdir:file1.txt where datdir is a logical name. At run-time, a program reads logical names table (this is Logntt00.iaf file at default directory) and creates absolute file name.By simple changing just one line in logical name table, you can redirect your file at any folder.

Record format of Logntt00.iaf file is:

DEFINE logical_name physical_folder

where is

  
DEFINE
mandatory keyword
logical_name
your logical name
physical_folder
absolute path. If the path contains blanks, physical_folder must be surrounded by quotation marks. There are two special folders:
. (dot) denotes current folder.
..\dir denotes a folder parallel to current folder.
 

Logntt00.iaf is an ordinary ANSI text file and can be created/modified by any text editor. Also, this file can be created by a program, or via Iaf dialog.

Examples:

DEFINE datdir   c:\mydata
DEFINE prtdir   c:\myreports
DEFINE prgdat "c:\program files\myappl"
DEFINE myfile  c:\mydata\file1.txt
DEFINE wrkdir  .
DEFINE imgdir  ..\images

File names, see File definition dialog (physical file name), can be:

datdir:file1.dat
prtdir:report1.prt
prgdat:prog1.exe
myfile
wrkdat:file.tmp
