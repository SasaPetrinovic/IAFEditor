PDF417 barcode
   


PDF417 is a two-dimensional bar code consisted of multiple rows. The symbol can store up to 1100 bytes, or 1800 ASCII characters or 2700 digits. The maximum recommended amount of data is 800 characters. To encode letters from any code page, data should be in UTF-8 format.


Encoding mode

            Binary mode allows encoding all 256 bytes (0 to 255).

            Text mode allows encoding all printable ASCII characters and some control characters: TAB, line feed and carriage return.


Error correction level

            Error correction algorithm allows the barcode to avoid various damages of the symbol.

            <auto> : error level will be calculated at run-time.
            0 - 8 : nine error correction levels. Recommended minimum value is 2.

Number of data columns

             <auto> : number of columns will be calculated at run-time.
            The barcode may have 1 to 30 data columns. Recommended maximum value is 20.

Number of rows

             <auto> : number of rows will be calculated at run-time.
            The barcode may have 3 to 90 rows.

            Warning: the combination number_of_columns=<auto> and number_of_rows=<auto> is not recommended. Usually, one parameter is defined and the other one is calculated.

Aspect ratio

            Aspect ratio is the ratio of the barcode height vs. width. If number of columns and number or rows have the fixed values, this parameter is ignored.

            Examples:

            Aspect_ratio = 2 ( 2 : 1)             width = 1/2 height
            Aspect_ratio = 1 ( 1 : 1)             width = height
            Aspect_ratio = 0,5 (1 : 2)           width = 2 * height
            Aspect_ratio = 0,33 (1 : 3)          width = 3 * height
            Aspect_ratio = <auto>               Aspect ratio is ignored. The encode function will generate minimum number of codewords.

Module width (X dimension)

            A module is the smallest unit of the barcode symbol.

Module height vs. width ratio (Y : X)

            Recommended minimum value is 3,0.

Left margin

            Left margin in millimeters. This is the distance from the left edge of the sourcefield object to the barcode.

Top margin

            Top margin in millimeters. This is the distance from the top edge of the sourcefield object to the barcode.

Compact PDF417

            To decrease bar code width, select this option. Right row indicator will be removed and stop character will be reduced to a single bar.


Example

Display characters from code page 1250

            $x = compress_trim(fld1, fld2, ...)
            $x = convert($x,1250,65001)      !! convert to UTF-8
            Source = $x

and select encoding_mode = BINARY.
