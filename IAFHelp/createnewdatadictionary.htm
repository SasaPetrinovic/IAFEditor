Create New Data Dictionary
   

A Data Dictionary is place (warehouse) which contains all the necessary information about the application(s).

Data Dictionary name

Data Dictionary name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $. The first character must be a letter.

Location

Location is directory (folder) for new Data Dictionary. You can click Browse button to select an existing directory or manually enter the path if the directory does not exist.

Application name

Application name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $. The first character must be a letter. The application name need to represent nature of job which should be processed, like PERSONNEL, PAYROLL, INVENTORY, etc.

Protect Data Dictionary by password

If you want to protect your Data Dictionary by password, select this check box. In that case, an user attempting to open the Data Dictionary must enter the password.

Password

The password is up to eight characters long.


Click Create DD.


If the directory does not exist, message box "Directory does not exist. Create it?" appears on the screen. Click Yes if you want to continue.


After that, new dialog Change Setup Parameters displays on the screen, and process of DD creation finishes.

As a result, the directory contains new eight files: ddname.DD1, ddname.DD2, ... ddname.DD6, ddname.DD7 and Logntt00.iaf. File Logntt00.iaf will contain logical file names definitions if we use flat files in our application(s). At this moment, this file is empty.
