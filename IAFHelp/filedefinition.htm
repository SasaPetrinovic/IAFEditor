File definition
   

Generally, a file is collection of information. In Iaf, a file is collection of records containing data.

File organization


·     Sequential (flat) files contain set of records.
·     Indexed (index-sequential) files besides data records contain indexes (keys) which enable fast access to the records. In this chapter we will no consider this type of file organization.

Record format


·     FIXED. Each record in the file has the same length. The fields in such record can be of any type (alpha, integer, floating, packed numeric, etc.). The access to this file can be sequential or random (by record number).

·     VARIABLE. Each record is ended by record terminator. In our case the terminator is two-byte one: carriage return/line feed. Carriage return has decimal value 13, line feed has value 10. Because of this, fields in such records must be alpha ones: no integers, no BCD packed, no floating numbers. Such files can be viewed by an ordinary text editor.

·     VARIABLE WITH FIELD DELIMITERS. Each record is ended by record terminator (<CR><LF>) and each field in the record is ended by field delimiter.

            Delimiter can be any printable character or <TAB> (TABulator has decimal value 9).

            Example, delimiter is #:

            ABC#100#20070505#xyz
            REC 2#200#20070506#xyz


·     VARIABLE WITH UNICODE UTF-16 TEXT. This record has the same characteristics as VARIABLE one, except that each character consists of two bytes. Byte-order mark, a special Unicode character (0xFEFF), is placed at the beginning of the file.


·     VARIABLE WITH UNICODE UTF-8 TEXT. This record has the same characteristics as VARIABLE one, except that each field defined as UTF-16 will be converted to UTF-8 before writing the record. Such fields have the same length in characters, therefore, record has variable length in bytes.


Note

            Files with variable record format can be accessed only sequentially (from the beginning to the end). Records in such files cannot be updated.

Physical file name

            This is a path identifying the file on the disk. The logical names can be used in file name. If we use logical names, a program at run-time determines true path via definitions in Logntt00.iaf file (see also Logical names table maintenance).

Examples:

            c:\temp\myfile.tmp
            \\server\c\dir\myfile.seq
            d:\dir\a.txt
            data_dir:myfile.seq

            In this case, one of lines in Logntt00.iaf file must be like this:

            DEFINE DATA_DIR c:\mydir

            and according to this definition, absolute path will be c:\mydir\myfile.seq.


Record name

            In the most cases, all the records in one file have the same structure and we need to define this structure by selecting one of defined records.


Use multiple record definitions

            In a very few cases, one file contains records with different structures. Iaf supports up to 8 different record structures (redefinitions) in one file. To enable multiple redefinitions, select this check box. After that, we need to select all redefinitions, maximum 8.

Record identification

            Record identification must be entered in the form FIELD_NAME = literal where FIELD_NAME is one of the fields with the same characteristics at the beginning of each record.

            Example:

            Record_1, identification ->         FIELD_1 = '01'
            Record_2, identification ->         FIELD_1 = '02'
            Record_3, identification ->         FIELD_1 = '99'

            This means the following: all the records which have 01 in the FIELD_1 have structure of Record_1, etc. According to this, in Program Generator we can write:

      read_all myfile
            if field_1 = '01'
                  xxx = myfile.record_1.field_1
                  yyy = myfile.record_1.field_2
            else_if field_1 = '02'
                  xxx = myfile.record_2.field_1
                  aaa = myfile.record_2.field_2
                  bbb = myfile.record_2.field_3
                  ccc = myfile.record_2.field_4
            else_if field_1 = '99'
                  xxx = myfile.record_3.field_1
                  ppp = myfile.record_3.field_2
                  qqq = myfile.record_3.field_3
                  rrr = myfile.record_3.field_4
            end_if
      end_read


To write records into such file, just fill all the fields in record structure and call WRITE:

myfile.record_1.field_1 = '01'
myfile.record_1.field_2 = 'Abcdefghijklmn'
Write myfile

myfile.record_2.field_1 = '02'
myfile.record_2.field_2 = 152
myfile.record_2.field_3 = today
myfile.record_2.field_4 = systime
Write myfile

myfile.record_3.field_1 = '99'
myfile.record_3.field_2 = 152
myfile.record_3.field_3 = 1200,56
myfile.record_3.field_4 = 100,45
Write myfile


Program generator does not control the field_1 contents. We can write

myfile.record_1.field_1 = 'XX'
myfile.record_1.field_2 = 'Abcdefghijklmn'
Write myfile

but in this case the record structure will be undefined.
