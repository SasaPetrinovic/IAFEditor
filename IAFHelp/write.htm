WRITE
   

This statement writes record into a file (table).

Format 1 writes record into a file


            WRITE[/NOTRUNCATE] file[.record]

Format 2 inserts row into RDB table

            WRITE [catalog.][schema.]RDBtable [ [OMIT] (field1, …, fieldN)]

Where


·     file - the name of the file from datasource dialog. Access to the file has to be WRITE, WRITE_NEW or UPDATE.
·     record - record name if the file has many record redefinitons.
·     schema - database schema, optional. If omitted, current schema is used.
·     RDBtable - the name of the RDB table from datasource dialog. By default, all the fields will be inserted into the table. To insert specified fields into the table, use list of fields. If some fields should be omitted from INSERT command, use OMIT keyword with list of fields.



Qualifiers

            /NOTRUNCATE can be used for sequential files with variable record format. In that case, trailing blanks will NOT be removed from the record buffer. By default, trailing blanks are removed from the buffer.

Note

            The second format of the instruction inserts specified fields into the RDB table. The generator converts this statement into the appropriate SQL one:

INSERT INTO rdbtable (field1,field2, ..., fieldN) VALUES (:val1, :val2, ... :valN)

            If list of fields is omitted, all fields are included in INSERT.

            
Examples

  
! Files
      WRITE filename          !! standard format
      WRITE filename.rec1     !! file with many record redef.
      WRITE/NOTRUNCATE filename     !!do not remove trailing blanks

! RDB tables
      WRITE rdbtable          !! insert all fields into the table
      WRITE rdbtable (fld1,fld2) !!insert two fields only: fld1 and fld2
      WRITE rdbtable OMIT (flda,fldb) !!insert all fields except flda and fldb
      WRITE schema1.rdbtable  !!insert all fields, schema=schema1
      WRITE catalog.schema.rdbtable
      WRITE "name with blank"
      WRITE owner."name with blank"
      WRITE "owner with blank"."name with blank"
      WRITE catalog.owner."name with blank"
