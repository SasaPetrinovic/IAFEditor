Source_field properties
   

Object name

When the new source_field is created, generator automatically assigns the name Sourcefield_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Member of group (information only, cannot be changed)

If the object is a member of a group, the name of this group is displayed. An object can be the member of one group only.

Source field style

  
Standard
Display text.
Image
Display image. The variable contains path to the image file, like c:\image\img1.bmp or my_images:img1.bmp. Allowed types: ICO, BMP, JPEG, GIF.
Barcode_XXX
Draw barcode. The variable contains barcode string. See Barcode printing.

Visible

Determines whether the object is visible when the subform is first displayed.

Display entire text, no clip

By default, text is clipped to the rectangle. The part of the text that is inside the rectangle is visible. The remaining part is invisible (transparent).
If we want to display entire text, regardless of clipping rectangle, this option should be checked. In this case, enough space must be reserved to the right of the object.
Example: an alpha field has edit mask "XXXX". Auto size (alignment option in painter) defines clipping rectangle according to this edit mask. At run-time, if the field contains "MMMM", only the part of the text will be seen, because the width of "MMMM" is greater than width of "XXXX". But, if we check this option, every field contents will be properly displayed.

Shrink to fit

Specify that whole text fits within given rectangle. If it is necessary, the font size will be reduced. Valid if multiline (see below) was not set.

Multiline (alpha fields only)

Specify that the field be displayed in multiple lines. Word wraping is on. If it is necessary, vertical scroll bar appears on the screen.
If the source_field is placed on the report form, the height of rectangle must be suitable. Otherwise, part of the text will not be seen (see below: Variable height).

Vertical scroll

Shows vertical scroll bar in multiline source field on the screen. Valid if multiline set.

Variable height (reports only)

This attribute is applied only if multiline is set. If this attribute is set, height of the object will be
determined so that the whole field is seen.

Split always across pages (reports only)

This attribute is applied only if multiline and variable_height are set.

If this check box is set, and there is no enough space on the current page, the first part of the field will be printed on the current page, and remainder will be printed on the next page.

If this check box was not selected, the algorithm is the following: if there is enough space on the current page, the field will be printed on the current page. If there is no enough space, whole field will be printed on the next page.

Note: if the field height is larger than one page, the field will be always split across pages.

Excel output (reports only)

If this attribute is set, field contents will be written in the output Excel table (see ...).

Background style

Values:
  
Normal
Background is filled with background color
Transparent
Color of the subform or report behind the object is visible.

If you plan to use the Display method with this object, background style should be set to normal, otherwise you can choose between normal and transparent.

Background color

This attribute is applied only if the background style is normal. Double click brings up a color definition dialog.

Special effects

Values:

Flat
Raised
Sunken
Etched
Bump
Shadowed

Border style

  
Transparent
border is not seen
Solid
the border is standard line
Dots
the border is dotted line
Dashes
the border is dashed line
Boxes
each character will be displayed in the separated box
Grid
like above: top of the box does not exist
Halfgrid
like grid: height of vertical lines is half height of the object
Grid transparent
like grid, but grid lines are invisible

Border color

Is applied only if border style is not transparent.  Double click brings up a color definition dialog.

Border width

Is applied only if border style is not transparent. Possible values are 0, 1, 2,... pixels.

Text color

Color of text. Double click brings up a color definition dialog.

Font name

Choose one from offered font names.

Font size

Specifies font size of the text.

Bold

Specify that font weight is 700. Normal weight is 400.

Italic

Specify whether text is italic.

Underline

Specify whether text is underlined.

External underline

External underline is line which is positioned 1 pixel below the object and extends up to the right end of the object. Let us mention that the font attribute 'underline' (see above)  draws line which extends up to the end of the text.

Tight horizontal

This parameter is valid only if the border style is solid, dots or dashes.
In that case, border surrounds text. If this check box is cleared, border surrounds whole object.

Orientation

Orientation of text in degrees. Horizontal text has orientation 0.
Possible values are between -90 and 90 degrees.

Horizontal text alignment

Values:

Left                  trim left blanks then left align
Center
Right
None                left alignment without trimming left blanks

Numeric fields are usually right aligned; alpha and date ones are left aligned.

Vertical text alignment

Values:

Top
Bottom
Center

Sequence number

See destination_field sequence number.

Tooltip text

Tooltip text is a text that will be displayed when the cursor hovers above the field. See Tooltip text definition.

Shading style

Valid if background style is Normal. Specifies that Color_from fades gradually to Color_to.

Values:

None                No shading
Horizontal
Vertical

Shading variant

Valid if shading style is Horizontal or Vertical.
Possible values are 1 to 4.

Color from

Starting color. Valid if shading style is Horizontal or Vertical.

Color to

Ending color. Valid if shading style is Horizontal or Vertical.
