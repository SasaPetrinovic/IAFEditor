Enablenewlines
   

This method enables or disables new line entering into a grid.

Format

            Enablenewlines(value)

Parameters


·    value - a numeric variable or numeric literal. If value = 0, user can modify existing lines only. If value <> 0, user can enter new lines.


Applied To

            Data entry table (grid).

Note

            By default, new lines can be entered.

            This method is used usually in 'Table starting actions'.


Example

  
            Table_1.Enablenewlines(0)     !! Disable new lines
            Table_1.Enablenewlines(1)     !! Enable new lines
!! Lets look at the following piece of code

!! Table starting actions
      select_all/load_key * from tbl1 where fld1 = :$Par1
      end_select
      Table_1.Enablenewlines(0)
!!
Loop SELECT_ALL ... END_SELECT reads record set from TBL1, simultaneously building internal table of primary keys (qualifier /LOAD_KEY). Run-time engine uses this internal table to properly display grid lines.
Method ENABLENEWLINES(0) disables entering new lines displayed into the grid.

At the end of 'Table starting actions', run-time goes through the internal table and displays grid lines, one by one. After the last line is displayed, 'Tables ending actions' will be activated, if exist, and finally, the cursor will be positioned in the first destination field of the first grid line.

The command Table_1.Enablenewlines(1), this is the default, positions the cursor to the first EMPTY line.
