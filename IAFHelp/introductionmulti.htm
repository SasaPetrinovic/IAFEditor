Using multiple datasources - Introduction
   

Suppose we have defined two datasources on our PC:

            DSN1 points to a MS SQL Server database. This is our main (default) data source.
            DSN2 points to a DB2 database. This is secondary data source.

            How to execute commands against tables on both databases in the same program?

            If our main database is MS SQL server, we could create a linked server to DB2 database and use one datasource only (DSN1). This is the simplest option.

            But, what if we cannot create linked server? Or if we use databases which do not have such functionality.

Let's consider two examples.

            Table TBL1 belongs to DSN1 (MS SQL Server).

            Table TBL2 belongs to DSN2 (DB2).

            Datasource DSN2 (DB2) has own table TBL1.
