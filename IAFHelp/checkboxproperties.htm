Checkbox properties
   

Object name

When the new checkbox is created, generator automatically assigns the name Checkbox_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Member of group (information only, cannot be changed)

If the object is a member of a group, the name of this group is displayed. An object can be the member of one group only.

Caption

The text which appears on the button. If you want to define one letter as a prefix (mnemonic) character, precede it with ampersand (&). At run-time, that letter will be underlined and user can click that button by pressing Alt + letter.
If you want to display single ampersand in the caption, enter two consecutive ampersands.

Within one subform, all prefix characters must be different.

Field name (information only, cannot be changed)

Name of the destination field. In IAF, data entered via keyboard must be recorded somewhere: into a table field, a local field, a counter, an accumulator. This place we call destination field.

Checked value (information only, cannot be changed)

The destination field can have two values only. When this value is 'checked_value', check mark on the screen will be set.

Unchecked value (information only, cannot be changed)

When the destination field contains 'unchecked_value', check mark on the screen will be cleared.

Visible

Determines whether the checkbox is visible when the subform is first displayed.

Enabled

Determines whether the checkbox is displayed as disabled when the subform is first displayed. When the checkbox is disabled, mouse clicks are ignored.

Background style

Background style should be set to normal.

Background color

This attribute is applied only if the background style is normal. Double click brings up a color definition dialog.

Special effects

Values:

Flat
Raised
Sunken
Etched
Bump
Shadowed

Border style

Values:

Transparent      border is not seen,
Solid                 the border is standard line,
Dots                 the border is dotted line,
Dashes             the border is dashed line.

Border color

Is applied only if border style is not transparent.  Double click brings up a color definition dialog.

Border width

Is applied only if border style is not transparent. Possibly values are 0, 1, 2,... pixels.

Text color

Color of text. Double click brings up a color definition dialog.

Font name

Choose one from offered font names.

Font size

Specifies font size of the text.

Bold

Specify that font weight is 700. Normal weight is 400.

Italic

Specify whether text is italic.

Underline

Specify whether text is underlined.

Horizontal text alignment

Values:

Left      Text left, checkbox right
Right    Text right, checkbox left

Sequence number

If the button has sequence number 0, click is allowed in every moment if the button is enabled. See Using objects with sequence number equal zero.

Tooltip text

Tooltip text is a text that will be displayed when the cursor hovers above the button. See Tooltip text definition.


Read only

When this attribute is set, checkbox cannot accept mouse click. Therefore, the checkbox behaves as usual source field, just has the graphic shape of checkbox.

Causes validation

See Button properties.
