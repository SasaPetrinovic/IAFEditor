Record definition
   

A record consists of fields. Maximum number of the fields in a record is 255.

To specify a field, just enter its name in Field name column. Field name is maximum 30 character long, allowed characters are: A - Z, 0 - 9, _ (underline) and $ (a dollar sign). Field name must not be the same as the record name. This name we will use later in our programs. In the next column (Global field name) select one of global (template) fields we already defined. The global field defines characteristics (type, size, data type, edit mask, etc) of 'Field name'.

Note: If we enter name of a global field in Field name column, the second column will be skipped.

If our field is a vector, enter number of occurences in the third column. Space (or value 1) in this column denotes an ordinary field. The last column (Key) applies to index-sequential files (Btrieve, etc.). Because we use database in our applications, this check box should be unchecked. This means that we create sequential files only. These files are usually used as working or temporary files.

Each field has local attributes (unlike global attributes defined by global field):
·     Column header (used in default programs in Form or Report painter).
·     Help text, maximum 80 characters, used in data entry programs.
·     Default value, a literal. This value will appear on the screen at run-time when an user try to enter value from the keyboard into this field. We can override this value by /DEFAULT qualifier in Form painter, see the Destination statement.
·     Range control in the form minimum ... maximum. This control executes after field entering. Examples: 'AAA' ... 'GGG' or ... 'KKK' or 'GGG' ...
·     Description, maximum 80 characters.



If a necessary global field does not exist, we can create it by clicking Define Field.

Detail characteristics of global fields we can see by clicking Show Fields.


How to delete a field?

Click side heading of the grid. Whole line becomes selected.
Press "Delete" key.

How to insert the field in the middle of the record

Click side heading of the grid. Whole line becomes selected.
Press "Insert" key.

How to change field position?

Click side heading and drag the field name down or up to the desired position.


Save & Exit

By clicking this button, record structure will be saved in Data Dictionary and text file recordname.clb will be created. This file contains record definition for C compiler.

If the new record was created, the dialog Default File Definition appears on the screen. We should remember that record defines just structure, while a file contains real data. If we click OK then new file definition will be stored to the DD. Default file parameters are:
·     File organization iz SEQUENTIAL if record does not have keys, or INDEXED if record has keys.
·     Record format is FIXED.
·     Physical file name is filename.SEQ if file sequential, or filename.IDX if file indexed.

These parameters can be changed later in File definition dialog.

If we modified the structure of an existing record, then Initialize / Reformat file dialog appears on the screen. We have two options: Initialize (create an empty file) or reformat (copy old records to the file with new record structure). Of course, we always can click Cancel to ignore these options. In this case, the file with old record structure stays on the disk.
