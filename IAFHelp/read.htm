READ
   

This statement reads one record from a file or RDB table by a key.

Format

READ [/nohold][/trim][/return_lock] file_name [PART_]KEY[n] = key [WHERE cond_exp]

Where


·     file_name - the name of the file (table) from datasource
·     key - concatenation. Number of segments in concatenation must be equal to the number of key segments
·     cond_exp - IAF conditional expression, which must have TRUE value


Qualifiers

            /NOHOLD reads record without locking if indexed file is open for update (UPDATE). While record is locked other programs can read it, but can’t erase or change it. Locked record is released after update or after the other record is being read.

            /TRIM reads record with partial key (only for alphanumeric segments of the key. Key size is determined by deleting all blanks at the end of the last segment.

            /RETURN_LOCK does not waits for releasing of locked record, but returns status immediately into the main program. Status can be checked with IF or CHECK (check record xxx not_locked).

Note

            This statement is mainly intended to use with index-sequential files like Btrieve.

            If we use this statement to read a record from a RDB table, /NOHOLD and /TRIM can not be used. This statement will be translated into standard SQL statement:

SELECT * FROM table_name Where ...

For RDB, SELECT statement is recommended.

            Also, the statement can be used to read random records from a sequential file with fixed-length record. In that case, KEY is relative record number:

READ SEQFILE KEY=5            !! Read fifth record

Examples

  
      READ tbl1 KEY='ABC'
      READ tbl2 KEY2=seg1 | seg2
