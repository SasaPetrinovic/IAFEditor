TRANSLITER
   

This function converts a text from Latin script to Cyrillic script and vice versa (Serbian language only).

Format

            TRANSLITER(alpha_field, conv_direction)

Return Value Type

            Alpha.

Parameters


·     alpha_field - field to be converted, alpha literal or alpha variable (ANSI or Unicode UTF-16).
·     conv_direction - conversion direction, alpha literal or alpha variable. Possible values are: "tocyril" and "tolatin".

            Tocyril  - convert Latin to Cyrillic
            Tolatin  - convert Cyrillic to Latin.


Notes

            Latin LJ / lj converts to ? / ? Cyrillic.
            Latin NJ/ nj converts to ?/ ? Cyrillic.
            Latin DŽ/ dž converts to ?/ ? Cyrillic.


Possible cases:


·           ANSI_result = transliter(ANSI_field,'tocyril'). ANSI_field has to contain a text in code page 1250. Result will be the Cyrillic text in code page 1251.
·           UTF16_result = transliter(ANSI_field,'tocyril'). ANSI_field has to contain a text in code page 1250. Result will be the Cyrillic Unicode text.
·           UTF16_result = transliter(UTF16_field,'tocyril'). UTF16_field has to contain a Unicode Latin text. Result will be the Cyrillic Unicode text.


·           ANSI_result = transliter(ANSI_field,'tolatin'). ANSI_field has to contain a text in code page 1251. Result will be the Latin text in code page 1250.
·           UTF16_result = transliter(ANSI_field,'tolatin'). ANSI_field has to contain a text in code page 1251. Result will be the Latin Unicode text.
·           UTF16_result = transliter(UTF16_field,'tolatin'). UTF16_field has to contain a Unicode Cyrillic text. Result will be the Latin Unicode text.


How to process UTF-8 fields?

            While UTF-16 characters have fixed length of two bytes, one UTF-8 character is represented by sequence from 1 to 6 bytes. Our generator recognizes only two types of alpha fields: single-byte fields (type A) containing a text in any code page and two-bytes fields (type W) containing an UTF-16 text, see Local fields definition. We can define a single-byte alpha field and move UTF-8 content to it. However, many string functions cannot recognize such type of text: str_length, upper, lower, etc. The simplest way to deal with UTF-8 fields is to convert them to UTF-16 ones using CONVERT function:
            $utf16 = CONVERT(alpha_field_containing_utf8_text,65001,1200)
From this point, we can apply all the string functions to $UTF16 field.

How to exclude some parts of text from conversion?

When you convert a text from Latin to Cyrillic, some parts of text should be stay in Latin. For example, Roman numerals. Such parts of text must be surrounded by two hash signs (# ).

Examples

  
      $a = TRANSLITER("abcdlj šđžčć",'tocyril') !! result is "????? ?????"
