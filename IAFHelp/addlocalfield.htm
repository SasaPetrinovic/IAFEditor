Add local field
   

In this dialog we can create one local field.

Local field name

Local field name is maximum 16 characters long, the first character must be dollar sing ($).
Allowed characters: A - Z, 0 - 9, _ and $.

Examples:

$LOC1
$LL
$ALPHA1
$ALPHA_1
$ALPHA$1
$AAAAAAAAAA12345678         invalid, name too long
$$ALPHA         invalid, the second character is $, see DEFINE_LOCAL

Field type

Select one of field types from combobox.

Alpha (ANSI)                 - single-byte alpha field
Alpha (Unicode)            - double-byte, UCS-2, alpha field. Number of bytes occupied by such field is 2 * number_of_characters.

Field size

For alpha fields, maximum number of characters, not bytes.

For date, allowed values are 8 or 6.

For time, allowed values are 6 or 4.

For numeric, field size is in the form NN[,DD] where is NN - total number of digits, DD - number of decimals. For instance, 5,2 means: total digits 5; integer part 3 digits; two decimals.

Data type

This parameter applies to numeric fields only.
Default is unpacked: one digit occupies one byte, the sign is combined with last digit.

Number of elements

If you want to define an array, enter number of elements. Maximum number of elements is 9999.

Show local fields

To see already defined local fields, click this button.
