OPEN
   

This statement opens a file or establishes connection to a database.

Format 1

            OPEN file

Format 2

            OPEN /DATABASE connection_string


Where


·     file - the name of the file from Define datasource dialog.
·     connection_string - ODBC connection string, alpha literal or alpha variable, in the form: odbc_datasource;UID=username;PWD=password. By default, a program automatically connects to a database at the very beginning of the program. But, if checkbox 'Open database manually' in 'Define datasource dialog' is checked, this command must be used to connect to the database.


Note

            All the files, except these ones with /MANUAL qualifier, are opened at the very beginning of the program. Only /MANUAL files have to be opened by OPEN statement later-on in the program. Physical file name will be taken from the file definition in the DD. If you want to override this name, use /SPEC qualifier. But, if you want to supply physical file name at run-time, use an assignment statement with FILE_NAME keyword on the left-hand side, see example below.

            In reports and batch programs, files with WRITE and WRITE_NEW access mode, will be opened (created) just after program title, and before the program cycle.

Example

  
! Use physical filename from DD
      OPEN file1

! Variable physical filename.
! Of course, in 'Datasource' dialog the qualifier /MANUAL is used
! with file1.
      file1.FILE_NAME = "c:\test\a.seq"
      OPEN file1

! Open database
      WHENEVER SQLERROR CONTINUE
      OPEN /DATABASE 'my_datasrc;UID=user1;PWD=mypassw'
      IF iaf_sqlerror <> space
            Pause 'Open database error ' | iaf_sqlerror
      END_IF

! Open database using local variable
      $cs = compress_trim('my_datasrc;UID=',$usr,';PWD=',$pwd)
      WHENEVER SQLERROR CONTINUE
      OPEN/DATABASE $cs
