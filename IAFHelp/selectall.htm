SELECT_ALL
   

This statement retrieves many records (rows) from a RDB table(s) in a cycle. The cycle ends with END_SELECT.

Format

            SELECT_ALL select_statement

Note

            If we use this statement in 'Table starting actions' to prepare internal tables in a data-entry grid, we must use /LOAD_KEY qualifier. In this case, all the fields of the primary key must be listed in select_list. To correctly order records in the grid, ORDER BY clause must be used.

            SELECT_ALL/LOAD_KEY select_statement
                        ...
            END_SELECT

The same goal can be reached by READ_ALL/LOAD_KEY, but SELECT_ALL enables the usage of complex selection criteria, joins, etc. If we use many tables in FROM clause, the first table is the main one.

            See also READ_ALL.

Example

  
      SELECT_ALL * FROM tbl1 Where PKID < 100 order by PKID
            ...
      END_SELECT

      SELECT_ALL * FROM tbl1 WHERE PKID < :$PARAM order by 1,2
            ...
      END_SELECT
