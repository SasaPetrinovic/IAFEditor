CORR
   

This function moves corresponding fields from one table (record) to another table (record).

Format 1

            output_record = CORR(input_record)

Format 2

            output_file.output_record = CORR(input_file.input_record)

Return Value Type

            None.

Parameters

·     input_record - input record

Note

            Corresponding fields are fields which have equal names and types in both tables.

The following table displays allowed combinations.
  
Source field
Destination field
 
 
ANSI alpha (char)
ANSI alpha (char)
Unpacked numeric
ANSI alpha (char)
 
 
Unicode UTF-16
Unicode UTF-16
 
 
Numeric
Numeric
Date/datetime (1)
Date/datetime
Time
Time
 
 

(1) Possible datetime combinations:
            date                  -> date
            date                  -> datetime
            datetime           -> date
            datetime           -> datetime


Examples

  
!           Table_1 descr                 Table_2 descr
!     fld1, type A                  fld1, type A
!     fld1x, type N                 fld2, type N
!     fld2, type N                  fld3, type D
!     fld3, type D                  fld4, type D
!     fld4, type T

      table_1 = CORR(table_2)

!     Result:
!
!           table_1.fld1 = table_2.fld1
!           table_1.fld2 = table_2.fld2
!           table_1.fld3 = table_2.fld3
!
!If file File1 has two record redefinitions (rec1, rec2)
!and File2 has redefinitions rec11 and rec12, we can use
!the following formats:

      File1.rec1 = CORR(file2.rec2)
      File1.rec2 = CORR(file2.rec1)
      File1 = CORR(file2)     !! defaults=rec1,rec11
      File1.rec1 = CORR(file2)!! default =rec11
      File1.rec2 = CORR(file2.rec2)
