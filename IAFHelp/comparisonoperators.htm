Comparison operators
   

Comparison operators are:

  
=
equal to
<>
not equal to
>
greater than
>=
greater than or equal to
<
less than
<=
less than or equal to
is [not] containing
Result is TRUE if an alpha field contains a text, case-sensitive.
Example:
IS containing 'abc'
is [not] date
Result is TRUE if an alpha field contains valid date in the form YYYYMMDD. If alpha field is 6 characters long, the date form is YYMMDD.
is [not] holiday
Result is TRUE if a date field is a holiday.
is [not] in
Result is TRUE if a field matches any value in the list. A list consists of items separated by semicolon. The item can be a field or a range in the form lower_limit ... upper_limit.
For example:
IS in (1; 3; 5; 100)
IS in (1; 3; 5 ... 80; 150 ...)
IS in ('A'; 'B'; 'F' ... 'H')
IS NOT in (50; 100 ... 200; 350; 500...)
is [not] in_masktable
 
is [not] matching
Result is TRUE if an alpha field matches a specified pattern, case-sensitive.

Special characters in pattern are:

*           one or more characters
?          any character
#          any digit
[charlist] any character in charlist
[!charlist] any character not in charlist

Examples:
IS matching '*abc*'
IS matching 'AB*'
IS matching '*AB'
IS matching 'AB*X'
IS matching '?abc'
IS matching '??###'
IS matching '[abdgABDG]*'
IS matching '[a-zA-Z][a-zA-Z0-9]##*'
IS matching '[a-zšžA-ZŠŽ]*'
IS matching '[!a-g]*'
is [not] matching_ci
Same as MATCHING, but case-insensitive.
is [not] numeric
Result is TRUE if an alpha field contains digits only. If you want to check just left part of the field, use RTRIM function.

Example: if an alpha field is 10 characters long, result is TRUE if the field contains ten digits.
is [not] numeric_literal
Result is TRUE if an alpha field contains valid numeric literal, see value expression.
is [not] null
Result is TRUE if a database field has NULL value.
During SELECT or another READ statement, field value will be moved to the appropriate buffer. Also, null flag will be stored into a special variable. By using this comparison operator, we actually check that special variable.
Note: only database fields have null flag. Flat files fields do not have such attribute.
is [not] starting_with
Result is TRUE if an alpha field starting with text, case-sensitive.
Example:
IS starting_with 'ABC'
 
 
RECORD recordname [NOT_]FOUND
Result is TRUE if previous READ or SELECT statement successfully read a record.
RECORD recordname [NOT_]DUPLICATED
Result is TRUE if previous WRITE or INSERT statement tried to write a record with existing primary key (or unique index).
RECORD recordname [NOT_]LOCKED
Result is TRUE if previous READ/return_lock statement tried to read already locked record. This option is available if a database has explicit row locking option, such as Oracle and SQL Server.
 
 
FILE filename [NOT_]FOUND
Result is TRUE if file 'filename' exists. If filename contains special characters *,?,|,<,>,& result is always FALSE.
Examples:
IF FILE 'c:\temp\xx.tmp' FOUND
IF FILE 'apli_dat:xx.tmp' FOUND
 
 


Examples
  
      if a = b OR a = c
      if a <> b
      if a is containing 'abc'
      if a is containing b
      if a is in (1; 5; 7; 9 ... 12; 15 ...)
      if a is not date
      if a is numeric
      if rtrim(a) is numeric
      if alltrim(a) is numeric
      if a is numeric_literal
      if a is not matching '*abc*'
      if a is matching_ci '*abc*'
      if a is not starting_with b

      read file1 key = $k1
      if record file1 not_found
            ...
      end_if

      write file1
      if record file1 duplicated
            ...
      end_if


      if file 'c:\mydat\abc.txt' found
            ...
      end_if

Note1:
      if a is containing 'abc' is the same as

      if a is matching '*abc*'

Note2:
      If you do not use literal in containing, matching, starting_with, run-time engine trims right spaces from the second operand and then performs comparison. Example: if local variable $b has the value 'ABC DE    ', then
      if field is containing $b is the same as
      if field is containing 'ABC DE'
