DB_TABLES
   


This function stores list of tables, views and synonyms into text file.

Format

            DB_TABLES(schema, output_file)

Result Value Type

            Numeric. If the function fails result is -1, otherwise, result is 0.

Parameters


·     schema - name of the schema, alpha literal or alpha variable. If you want to export tables from all schemas, enter empty string or *.

·     output_file - output file name, alpha literal or alpha variable. The name can be: absolute path, like c:\temp\tbl.txt, or path with logical name, like work_dir:tbl.txt. The record consists of the three fields: table name, table type (TABLE, VIEW, ALIAS or SYNONYM) and schema. Field delimiter is TAB (ASCII code 9). In DD, File definition dialog, select 'Variable with field delimiters' option for record format.


Note

            To use this function, your program has to be connected to a datasource. To enable datasource connection, specify at least one RDB table in Datasource dialog. Only tables from the current connection can be listed. If the program uses multiple connections, the first connection will be used for export. Username, used in connection string, must have privileges to read catalog information from the database.


Examples

  
      $status = db_tables("","c:\temp\alltables.txt")
      $status = db_tables("*","c:\temp\alltables.txt")
      $status = db_tables("myschema","workdir:mytables.txt")
