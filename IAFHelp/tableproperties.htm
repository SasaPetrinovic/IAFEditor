Table properties
   

Object name

When the new table (grid) is created, generator automatically assigns the name Table_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Grid data entry options

Valid if grid is 'data-entry', i.e. at last one column contains destination field, combo-box or check-box. There are three options: Update a table, update arrays and spreadsheet.

            Update a table

            If this option is selected, data will be stored into fields of a database table. The table must have primary key. Run-time engine uses internal memory table to store primary keys of the table. In 'Table starting actions' internal table must be initialized by

            SELECT_ALL/LOAD_KEY * from Tbl1 Where ...
            END_SELECT

This set of instructions creates internal memory table, stores primary keys into it and implicitly defines initial number of lines in the grid.
In 'Line ending actions', records are updated or inserted by

            IF RECORD Tbl1 FOUND
                        UPDATE tbl1
            ELSE
                        WRITE tbl1
            END_IF

If the database changes should be permanently committed at the end of every line, a COMMIT statement have to be added after END_IF.

            Update arrays

            If this option is selected, data will be stored into arrays. Each data entry column should contain an instruction like
            
            DESTINATION = $arrayN(CURRENT_LINE)

In 'Table starting actions' arrays should be filled by

            init counter_1, $array1(), $array2(), ..., $arrayN()
            SELECT_ALL * from Tbl1 Where ...
                        add 1 to counter_1
                        $array1(counter_1) = fld1
                        $array2(counter_1) = fld2
                        ...
                        $arrayN(counter_1)= fldN
            END_SELECT
            Table_x.Setnumberlines(counter_1)       !! mandatory

To process entered data, you should have somewhere, usually in click actions of the OK button, set of instructions

            counter_1 = 1
            do_while counter_1 <= Table_x.Getnumberrows()
                        fld1 = $array1(counter_1)
                        fld2 = $array2(counter_1)
                        fld3 = $array3(counter_1)
                        ...
                        do_something with data
                        ...
                        add 1 to counter_1
            end_do



            Spreadsheet style


Process grid as a field

If this option is checked, the grid will be processed as an ordinary, sequenced, field.


Auto click actions

Valid if grid is 'display-only'. If this option is checked, table click actions will be activated when the grid changes current line (using mouse click, arrow up, arrow down, etc.).

Visible

Determines whether the object is visible when the subform is first displayed.

Context menu

If checked, context menu commands can be created for this grid. These entries will be displayed together with fixed commands in the same popup menu.

Multiple selection

If checked, specifies that the user can select many lines at run-time using Ctrl+Click and Shift+Click keys. To unselect selected lines, context menu command 'Clear selection' can be used.

Disable column resizing

Specifies that width of columns cannot be changed at run-time.

Disable row dragging

Specifies that read-only grid rows cannot be dragged at run-time.

Multiline column headers

Normally, text in the top header will be displayed in the single line. If the text is wider than the column width, it will be cut off.
To display long text in multiple lines, set this checkbox. Of course, the height of the top header must be enlarged.

Suppress top heading

Set this checkbox if the top heading need to be invisible at run-time.

Suppress side heading

Set this checkbox if the side heading need to be invisible at run-time. If this option is selected, side heading and corner button disappear from the grid; therefore we cannot use corner button and side heading context menus. To enable this functionality, just click 'Unhide column(s)' command from the grid context menu, display corner button context menu, perform desired operation and click somewhere outside the grid.

Suppress vertical scroll bar

Set this checkbox if the vertical scroll bar need to be invisible at run-time. If the grid has fixed number of lines, for example 12, and subform has enough space to display such object, you could suppress side heading and vertical scroll bar. After that, in context menu, select 'Calculate exact grid height' command to calculate grid dimensions. This command is available if: side heading is suppressed, vertical scroll bar is suppressed and grid lines have fixed (not variable) height.


Check box style

Specifies check box style if the grid contains column(s) with check box.

Values:

Sunken
Raised
Flat

Check box mark

Specifies check box shape.

Values:

Check mark
Cross

Initial number of rows

Number of initial lines in the grid.

Variable row height

Valid if grid is 'display-only'. This option makes sense only if a source field or a button have the multiline attribute set on.

Maximum row height

Valid if variable_row_height = YES and one of columns contains sourcefield/image.
Row height will be determined by image height. If image height is greater than maximum row height, run-time engine sets row_height to maximum_row_height and use 'zoom' method to preserve image proportions.

Line divider

Normal
Raised
Sunken
Vertical
Horizontal
No divider

Line divider color

Line divider color, valid if line divider is normal, vertical or horizontal. If the line divider is RAISED or SUNKEN, line divider colors will be white/gray.

Scroll bar tracking

Not used.

Termination condition

Valid if grid is 'display-only'. When this condition is satisfied, filling of the grid stops and program skips to 'Table ending actions'.

Suppress line condition

Valid if grid is 'display-only'. When this condition is satisfied, the row will not be displayed in the grid, but all calculations in the fields are performed.

Sequence number

See destination_field sequence number.

Display "Loading ..." message

Set this checkbox if you expect a lot of lines, (for example, more than 1000), in the grid. Valid if grid is 'data-entry' one.

"Zebra" style

Valid if grid is 'display-only'.
