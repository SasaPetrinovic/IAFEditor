UPDATE
   

This statement updates existing record of a file (rdb table).

Format 1

            UPDATE file
            UPDATE rdb_table

Format 2

            UPDATE rdb_table [ [OMIT] (field_1, field_2, field_3, ...)]

Where

·     file - the name of the file from datasource dialog. Access to a file has to be UPDATE or UPDATE_NO_CREATE. Record must be previously read by READ or READ_ALL.

·     rdb_table - rdb table from datasource dialog. Iaf generates SQL command using all the fields, except blob ones:
            UPDATE TABLE rdb_table SET field1=value, field2=value, ... fieldN=valueN WHERE pk_seg1=value1 AND pk_seg2=value2 ... AND pk_segN=valueN

            In format 2, we can write the list of fields and Iaf will generate SQL command using just the fields from the list:

            UPDATE TABLE rdb_table SET field1=value, field2=value, ... WHERE pk_seg1=value1 AND pk_seg2=value2 ... AND pk_segN=valueN

            If some fields should be omitted from UPDATE command, use OMIT keyword with list of fields.

            In both cases, the record (row) must be previously read by READ, READ_ALL or SELECT statement. If we use SELECT statement, all the segments of the primary key must be in select_list. Therefore, tables with no primary key, cannot be modified by UPDATE command (see Execsql).

Note

            A sequential file with variable record format, cannot be updated by Iaf.

How to modify blob fields?

            To do this, use format 2:

            UPDATE rdb_table (blob_field)

            or use EXECSQL.


Example
  
!Update record in an indexed file
      READ file1 KEY="AAA"    !! read record by primary key
      if record file1 found
            fld1 = 100        !! change some fields
            fld2 = "ABC"
            fld3 = TODAY
            UPDATE file1
      end_if

!Update sequential file with fixed record length
            READ seqfile KEY=5      !! read 5th record
      if record seqfile found
            fld1 = 100        !! change some fields
            fld2 = "ABC"
            fld3 = TODAY
            UPDATE seqfile
      end_if

!Update row in RDB table
      READ rdbtable KEY="AAA" !! read record by primary key
      if record rdbtable found
            fld1 = 100        !! change some fields
            fld2 = "ABC"
            fld3 = TODAY
            UPDATE rdbtable
      end_if
!or
      SELECT * from rdbtable WHERE pkfld='AAA'
      if record rdbtable found
            fld1 = 100        !! change some fields
            fld2 = "ABC"
            fld3 = TODAY
            fld4 = NULL
            UPDATE rdbtable
      end_if

!Update row in RDB table with blob fields
      READ rdbtable KEY=... !! read row
      ...   !! modify fields
      UPDATE rdbtable !! update all fields except blob ones
      UPDATE rdbtable (blob_field) !!update blob field
      COMMIT
