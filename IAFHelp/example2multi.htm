Example 2 - Copy TBL1 (DSN1) to TBL1 (DSN2)
   

Data Dictionary actions:

Load TBL1 (DSN1) definitions into your Data Dictionary.
Table TBL1 from DSN2 cannot be loaded because table name in the DD must be unique.


Case 1: TBL1 (DSN1) and TBL1(DSN2) have the identical structures.

In 'Define datasource' dialog  we select TBL1 and use alias for TBL1(DSN2)

            TBL1, access=read
            TBL1_DSN2 /USE: TBL1 /DSN: DSN2;UID=user;PWD=pwd, acess=write

Select "Connect to multiple datasources" check box.

Program code could be

      reset tbl1_dsn2
      select_all * from tbl1
            tbl1_dsn2 = corr(tbl1)
            write tbl1_dsn2
      end_select
      commit


Case 2: TBL1 (DSN1) and TBL1(DSN2) have similar structures (some columns are the same, some columns are different).

In 'Define datasource' dialog  we select TBL1 and use alias for TBL1(DSN2)

            TBL1, access=read
            TBL1_DSN2 /USE: TBL1 /DSN: DSN2;UID=user;PWD=pwd, acess=write

Select "Connect to multiple datasources" check box.

In program code we use only the same columns

      reset tbl1_dsn2
      select_all col_1, col_2, ..., col_N from tbl1
            tbl1_dsn2.col_1 = ...
            ...
            tbl1_dsn2.col_N = ...
            execsql insert into tbl1_dsn2 (col_1, col_2, ..., col_N) values ...
      end_select
      commit


Case 3: TBL1 (DSN1) and TBL1(DSN2) have different structures.

In this case we must create a synonym, alias or view in DSN2 under different name and load the synonym into the DD. After that, the procedure is the same as in Example 1.
