DEFINE_RECAP_RECORD
   

This statement writes one record into the recapitulation file.

Format

            DEFINE_RECAP_RECORD
                        RECAP_1 = field1
                        ...
                        RECAP_N = fieldN
            END_RECAP_RECORD

Where


·     field - value to be stored into the recapitulation record.

Note

            The block of the statements DEFINE_RECAP_RECORD ... END_RECAP_RECORD fills and writes one record into the recapitulation file. All the field names in this record must start with RECAP_. When the last record from main input record stream is processed, the recapitulation file closes and reopens in read-only mode, becoming the main input file in recapitulation report.

            Usually, we place the block within a field in subtotal calculations, because the recapitulation displays summary data. But, it is not necessary. If we use this block inside a detail line field, one record from input record stream will generate one record in recapitulation file.

            If we place DEFINE_RECAP_RECORD ... END_RECAP_RECORD block inside a do_while loop, one recapitulation record will be written for each loop pass.


Example

  
      DEFINE_RECAP_RECORD
            RECAP_1 = acc_1
            RECAP_2 = fld1 + fld2
            RECAP_3 = inptbl.field1
            RECAP_4 = total(inptbl.ammount)
      END_RECAP_RECORD
