Break fields definition
   

A break is an event which causes printing of a lineset in a report, or activation subtotal calculations in a batch program.

Break field is a field whose change activates the break, ie. on any change of this field, one of linesets will be printed.

Break field can be defined for page header, subheaders and subtotals.

Generally, the break field is a concatenated expression in the form

            field_1 | field_2 | field_3 | ... | field_N

where field_1, ..., field_N belong to the tables participating in view definition. Change of any segment activates break.

Break fields are tightly connected to sort order of input table(s). Sort keys from ORDER BY clause are candidates for break field segments.

Let us look at the next view definition:

Select * from inptbl order by CUSTOMER_ID, ORDER_NO

If we define subheader break field as

            CUSTOMER_ID

then we can display customer_id in the subheader, while the orders (sorted by order_no) can be printed in the detail line.



Using functions in break field

Allowed functions are: YEAR, MONTH, QUARTER, WEEK_OF_YEAR, LOWER and UPPER.

Example:

      field_1 | field_2 | ... | week_of_year(date_field)

This break will occur every Monday.



Using local fields in break field

If we want to use a local field in break field definition, the local field must be filled-up in 'Additional view actions'. After that we can use this field:

            Break_field = field_1 | field_2 | ... | $local_field

            -or-

            Break_field = $local_field

            if $local_field represents a concatenation in the form

                        $local_field = field_1 | ... | field_N
