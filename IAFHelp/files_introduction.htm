Introduction
   

To manipulate with operating system files, we use an object named FILE. This object enables us to read and write ordinary text files. Text files contain plain, unformatted text. Each record of a text file is ended by two-bytes terminator: carriage return/line feed.

All methods return an integer value.

There is only one FILE object in a program. Consequently,  two files cannot be opened at the same time.

Note
            If we want to define record structure, we must do it using 'Data Dictionary' utility. Such structured files must be declared in  Datasource dialog and processed using statements like WRITE, READ_ALL, etc.


Object name:

            File

Methods:

            Open - opens/creates a text file
            Close - closes the file

            Rewind - positions the file pointer
            Reset - resets file (removes all the records)

            Read - reads next record from the file
            Write - writes one record to the file
