Constants
   

Constants are predefined fields whose values cannot be changed, i.e. these fields cannot be used on the left side of an assignment.


BEGINCM - date, beginning of the current month
BEGINCY - date, beginning of the current year
BEGIN_Q1 - date, beginning of the 1st quarter of the current year
BEGIN_Q2 - date, beginning of the 2nd quarter of the current year
BEGIN_Q3 - date, beginning of the 3rd quarter of the current year
BEGIN_Q4 - date, beginning of the 4th quarter of the current year
BEGIN_QC - date, beginning of the current quarter

CRLF - carriage return/line feed pair, alphanumeric constant, 2 bytes, hex value = 0D 0A

ENDCY - date, end of current year
END_Q1 - date, end of 1st quarter of the current year
END_Q2 - date, end of 2nd quarter of the current year
END_Q3 - date, end of 3rd quarter of the current year
END_Q4 - date, end of 4th quarter of the current year
END_QC - date, end of the current quarter

NBSP - non-breaking space, used in COMPRESS_TRIM only
NULL - null value, can be used to assign null value to a RDB table field

PI - Ludolph's number, value is 3,14159265358979
P_MODE - alpha, program mode ("I" - main program, "S" - subprogram called by call_program)
PROGRAM_GENERATED - alpha, date of program generation in the form yyyy-mm-dd hh:mm:ss
PROGRAM_MODIFIED - alpha, date of last modification in the form yyyy-mm-dd hh:mm:ss
PROGRAM_MODULE - alpha, name of executable file
PROGRAM_NAME - alpha, contains program name
PROGRAM_VERSION - alpha, contains program version

SPACE - alpha, all spaces
SYSTIME - time, contains current time in the form hhmmss

TIMESTAMP - datetime, contains current date/time
TODAY - date, contains current date
TOMORROW - date, contains day after TODAY
TZ_VALUE - alpha, time-zone value in the form +-HH:MM, for example -02:00 or +03:00

USERNAME - alpha, username
USERID - alpha

YESTERDAY - date, contains day before TODAY

ZERO - numeric, all zeros
ZERO_DATE - date, all zeros
ZERO_TIME - time, all zeros
