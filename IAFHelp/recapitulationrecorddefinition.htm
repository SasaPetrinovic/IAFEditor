Recapitulation record definition
   


Using recapitulation file, we have possibility to print two reports in one program. The mechanism is the following: during main pass of the report, we write records into the recapitulation file. At the end of the main report, recapitulation report starts with the recapitulation file as the main one.
Usually, we store some totals in the recapitulation record, but it is not necessary. Everything we store in this file will be printed in the recapitulation report.

Maximum number of the fields in the recapitulation record is 50.

Field attributes: name, type, size

To specify a field, just enter its name in the first column. Field name is maximum 30 characters long starting with RECAP_, allowed characters are: A - Z, 0 - 9, _ (underline) and $ (a dollar sign).

In the next column select appropriate field type, and in the third column enter its size. For alpha, UTF16, date and time field, the size is an unsigned number. Numeric field size is in the form nn[,dd] where is

nn - total number of digits, maximum 18,
dd - number of decimals.


Example

  
RECAP_1
Alpha
10
RECAP_2
Num
11,2
RECAP_3
Num
5
RECAP_4
Date
8
RECAP_SUM
Num
15,2


How to delete a field?

Click side heading of the grid. The whole line becomes selected.
Press "Delete" key or right click to display context menu.

How to insert a field in the middle of the record

Click side heading of the grid. The whole line becomes selected.
Press "Insert" key. A new, empty line will be open.

How to change field position?

Click side heading and drag the field name down or up to the desired position.


Context menu

Right click displays context menu with Delete and Insert field commands.


Save & Exit

Click this button to save the modifications.

Generate 10 new fields

This command creates 10 fields, default name = RECAP_NN, field type = alpha, length = 10.

Include recapitulation record

This command activates the dialog with list of programs having recapitulation. Select one of programs and click OK to load its recapitulation record.

Delete recapitulation record

Click this button to remove recapitulation record. After this action, you have to remove define_recap_record ... end_recap_record block from your code and to delete recapitulation report, as well.


Note

This dialog just defines recapitulation record structure. To write records into the recapitulation file, use define_recap_record ... end_recap_record.
