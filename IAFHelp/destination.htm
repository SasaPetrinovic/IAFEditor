DESTINATION
   

This statement defines data entry field. The destination field object gets the input focus allowing user to enter field via the keyboard.

Format

            DESTINATION = field
                        [/ASSVAL: assval_expr]
                        [/AUTO_RETURN]
                        [/CHECKED: val_exp]
                        [/UNCHECKED: val_exp]
                        [/CALENDAR]
                        [/COLORDIALOG]
                        [/COND: cond_exp]
                        [/DEFVAL: val_exp]
                        [/DEFVALF: val_exp]
                        [/FILEDIALOG: Title, Initial_directory,  Filter (file_extensions)]
                        [/FONTDIALOG]
                        [/HELP: help_name or alpha_literal]
                        [/HELPA: help_name or alpha_literal]
                        [/LIST: list_name]
                        [/LOOKUP: SQL_statement [HEADING alpha_literal] [TITLE title]]
                        [/LOOKUPA: SQL_statement [HEADING alpha_literal] [TITLE title]] or
                        [/LOOKUP: lookup_name]
                        [/LOOKUP_ONLY: lookup_name]
                        [/LOOKUPC: lookup1, lookup2, ..., lookupN cond1, cond2, ..., condN
                        [/MISSING]
                        [/NO_ECHO]
                        [/NO_EXIT]
                        [/PROGRAM: "program_name"]
                        [/PROGRAM_ONLY: "program_name"]
                        [/RIGHTFILL[:character]]

Where

·     field - the field to be entered. What we enter via the keyboard, will be stored into this field.
·     cond_exp - conditional expression. If TRUE, the field will be entered manually, otherwise value from ASSVAL will be moved to the field and the next object in the sequence becomes the active object.
·     val_exp - value expression.
·     list_name - the name of the list defined in the list file ( IAFLOV.IAF). In lists are located fixed codes and descriptions.
·     program_name - the name of the program generated by IAF. It is an alphanumeric literal. Could be called upon by pressing CTRL/A
·     character - an alphanumeric character for filling the left, empty part of the field. Default character is blank.


Qualifiers

            /CALENDAR enables the user to select a date from the calendar dialog.

            /COLORDIALOG enables the user to select a color from the color dialog. The destination variable can be an alpha or numeric one. If we use alpha variable, it must be at least 11 characters long and result will be in the form R,G,B. If numeric variable is used, result is a number in the range 0 (black) to 16777215 (white). To get color components from this number, we have to use some arithmetic. If $RED, $GREEN and $BLUE are local three-digit numeric variables, and COLOR_VAR is the destination variable, the components could be calculated as:
$RED = MOD(color_var, 256)
$GREEN = MOD((color_var / 256), 256)
$BLUE = IDIVIDE(color_var, 65536)

$RED, $GREEN and $BLUE are in the range 0 to 255.

            /COND: cond_expr /ASSVAL: assval_expr
            These two qualifiers must be used either both or neither. We can read this as the following: if the condition cond_expr is satisfied (TRUE), the field will be entered manually, else assval_expr will be moved to the field and control will be transferred to the next field in the sequence.

            Destination = $numfield/COND: counter_1=1 /ASSVAL: 10

            If counter_1 contains 1, the field will get input focus, enabling data entering via keyboard.
            If counter_1 is not equal 1, value 10 will be moved to $numfield, this value will be displayed on the screen and control will be transferred to the next field.

            /DEFVAL: val_expr
            When the field is about to get input focus, program displays val_expr on the screen, expecting data entering. The user can press RETURN or TAB to accept this value, or can re-type it. This option applies only if the field is initialized (was not entered). If the field is already entered and you move the cursor to it, by mouse or keyboard, DEFVAL value will not be displayed on the screen.

            /DEFVALF: val_expr                (DEFVAL forced)
             When the field is about to get input focus, program displays val_expr on the screen, accepts this value and continues.

            /FILEDIALOG enables the user to select a file or a folder. The name of selected file or folder will be moved to the destination field. Of course, the destination field must be an alpha one. This qualifier activates one of two dialogs: file open dialog or 'browse for folder' dialog.

            If we want to select a file, the format is

            /FILEDIALOG: title, initial_folder, filter

            If we want to select a folder, the format is

            /FILEDIALOG: title, initial_folder, "", "D"

            title - dialog title, alpha literal or alpha field.
            initial_folder - initial file directory. Contents of this directory will be displayed after dialog activation.
            filter - alpha variable containing pairs of filter strings. The first string of the filter is description ("Image files", "Text files", etc.). The second string specifies filter pattern (*.bmp).
Pattern separator is semicolon: (*.bmp;*.ico;*.gif). The separator between two filter strings is |.
End of the filter is specified by two | signs (||).

Examples:

            $filter = "Image files (*.bmp) | *.bmp||"
                        filter description = Image files (*.bmp)
                        separator          = |
                        filter pattern       = *.bmp
                        end of filter        = ||

            $filter = "Image files (*.bmp,*.ico,*.gif) | *.bmp;*.ico;*.gif||"
                        filter description = Image files (*.bmp,*.ico,*.gif)
                        separator          = |
                        filter pattern       = *.bmp;*.ico;*.gif
                        end of filter        = ||

            $filter = "Text files (*.txt) | *.txt| All files (*.*) | *.*||"
                        filter description = Text files (*.txt)
                        separator          = |
                        filter pattern       = *.txt
                        separator          = |
                        next pair descr. = All files (*.*)
                        separator          = |
                        next pair pattern= *.*
                        end of filter        = ||
                        
            $filter = "Text files (*.txt) | *.txt||"

            Destination = $alpha/FILEDIALOG: "My files", "c:\tmp", $filter

            Destination = $dir/FILEDIALOG: "Devices", "My Computer", "", "D"
            Destination = $dir/FILEDIALOG: "Diskc", "c:\", "", "D"

            /FONTDIALOG enables the user to select font. The destination variable must be an alpha one. The result is a semicolon separated field in the form font_name;size;isbold;isitalic;isunderline.

            Example: Arial;12;1;0;0  (Name=Arial, size=12, bold, not italic, not underline)

            /HELP, /HELPA displays help text just below the destination field. If you use /HELP qualifier, F1 key must be pressed to display the help. If you use /HELPA, help text will be displayed automatically when the field gets input focus.

            Destination = $alpha/HELP: "Enter value between 10 and 100"
            Destination = $alpha/HELPA: "Enter value between 10 and 100"

If you want to display a small window with help text, create internal help screen in the painter, Define help screen command, and then use its name in the statement.

            Destination = $alpha/HELP: myhelp

            /LOOKUP, /LOOKUPA enables user to display some rows from a database table. If the user selects one line in the list box and clicks Move button, value from the first column will be moved to the destination field, see Lookup dialog.

                        Destination = $fld/LOOKUP: LOOK1

                        where LOOK1 is name of lookup dialog.


            /LOOKUPC - conditional lookup enables user to display different lookup dialogs.

                        Destination = $fld/LOOKUPC: look1, look2  $x=1, $x=2

                        where LOOK1 and LOOK2 are lookup dialogs, see Lookup dialog.
                        LOOK1 will be displayed if $x is equal to 1. LOOK2 will be displayed if $x is equal to 2.

            /LOOKUP_ONLY - cannot enter field interactively. Click magnifier to display a lookup dialog.

            /NO_ECHO enables user to enter a field without echoing on the screen.

            /PROGRAM: program_name enables user to call a program by clicking magnifier button or pressing CTRL+A. "Program_name" can be alpha literal or alpha variable. Command line parameters can be used, too.

            Destination = $fld/program: "myprog"
            Destination = $fld/program: "myprog.exe"
            Destination = $fld/program: "prog_dir:myprog"

            $varprog = "prog_dir:prog1"
            Destination = $fld/program: $varprog

            Destination = $fld/program: "myprog.exe p1 p2 p3"
            Destination = $fld/program: "myprog.exe PARAM=123"

            /PROGRAM_ONLY: program_name cannot enter field interactively. Click magnifier to select a value.

            /RIGHTFILL[:character] - shift right and left-fill by "character". If alpha field is not fully entered, it will be shifted to the right. Left, empty, part of the field will be filled by "character". Default value for "character" is blank.

            Destination = $alpha_field/RIGHTFILL: '0'

            If $alpha_field has length=5, and if you enter "123", the result will be "00123".

            Destination = $alpha_field/RIGHTFILL

            In this case, the field will be left-filled by blanks.


Notes

            The destination statement cannot be used inside a IF ... END_IF block. To define conditional data entry, use /COND and /ASSVAL qualifiers.


Examples
  
      Destination = fld1
      Destination = fld1/NO_ECHO
      Destination = fld1/DEFVAL: 50
      Destination = fld1/DEFVAL: acc_1
      Destination = fld1/HELP: "This is a numeric field without sign"
      Destination = fld1/PROGRAM: "apli_exe:prog1"
      Destination = fld1/COND: prev_fld > 0 /ASSVAL: 100
      Destination = fld1/LOOKUP: Select f1,f2 from TBL Where f2 > 100   
      Destination = fld1/LOOKUP: MyLookup  !! Using predefined LOOKUP entity
