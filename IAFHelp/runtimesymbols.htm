Run-time symbols
   

Using symbols, the behavior of Iaf programs can be changed at run-time.
All symbols are placed in the system Registry, under the key HKEY_CURRENT_USER\Software\Sing\Iafw\Rt as string values.

Symbols can be created manually using REGEDIT program. Also, IAFUTIL can be used.
To define a run-time symbol in Iaf program, we use WRITE_EXTERNAL statement where symbol name starts with a dollar sign:

WRITE_EXTERNAL/NAME: $symbol_name symbol_value

Examples:

Write_external/name: $iafmenun "mymenu.mnu"
Write_external/name: $iafworkd "c:\appl"
Write_external/name: $odbc_trace "1"

ACTIVECOLOR

This symbol specifies border color of active (current) field.
If the symbol does not exist, border color is red.
If the symbol has value -1 (minus 1), there is no active border coloring.
In other cases, symbol has the RGB format: RED,GREEN,BLUE  where red,green,blue components are in the range 0 - 255.

Examples
  
255,0,0
red
0,255,0
green
0,0,255
blue
0,0,0
black
255,255,255
white
-1
no coloring


ESC_ALLOWED

Normally, a user can use the ESC key to exit current program. If ESC_ALLOWED is defined and has the value 0 (zero), ESC can't be used.


DEFAULT_FONT

Font used in "List" and "Lookup" dialogs.

Format

            font_name, font_size [,codepage_script]

If this symbol does not exist, default is "MS Sans Serif, 8, 0".


Examples

  
Symbol value
Meaning
Arial,14,238
Arial, size=14, codepage=Central Europ.
Arial,14,204
Arial, size=14, codepage=Cyrillic
Arial,14,0
Arial, size=14, codepage=Western


DSN_applname

ODBC datasource used to connect to a database.
Normally, a program uses datasource defined in the program to connect to the database. But, if this symbol exists, datasource defined by this symbol will be used to connect to the database.
For example, if application name is MYAPPL, program looks for DSN_MYAPPL symbol.
If this symbol exists and its value is "MY_DSN",  program will attempt to connect to database via MY_DSN datasource. Of course, ODBC connect dialog will appear on the screen if it is necessary to enter some missing parameters, such as username, password, etc.

Also, this symbol can contain full connection string, for instance,

MY_DSN;UID=user1;PWD=pwd1

In this case, ODBC connect dialog will not appear on the screen.


IAF_HOLIDAYS

List of holidays. Holidays are used in NEXT_WORKING_DAY function.
All fixed holidays has the form: MMDD (four digits).
Variable holidays has the form: #YYYYMMDD (hash sign + 8 digits).
This symbol can be defined using IAFUTIL.

Example: if we have two fixed holidays: 1-jan, 2-jan and two variable ones: 31-mar-2006 and 4-jul-2006, the symbol has the value

            01010102#20060331#20060704


LANGUAGE_applname

In multilingual applications specifies current language: 1 for the first language, 2 for second, etc. If this symbols does not exist, a menu interpreter (Iafmenun, Iafmenui, Iafmenus) sets its value to 1. Later on, current language can be changed: right click activates "Change language" dialog.

Example: if application name is MYAPPL, program looks for LANGUAGE_MYAPPL symbol.


ODBCLOV

If this symbol is defined and has the value 1, list of values are placed in database. In all other cases, lists are placed in IAFLOV.IAF file.

ODBCLOV_PREFIX

If lists of values are in database (ODBCLOV = 1), we can define a prefix to enable all users access to these lists.

Examples:
If we use MS SQL Server, and lists are in MYLISTS database, owned by DBO, symbol value is "MYLISTS.DBO.".

If we use Oracle, and lists are in MYLISTS schema, symbol value is "MYLISTS.".

ODBC_TRACE

If the symbol is defined and has the value 1, all database activities will be recorded into ODBCTRAC.LOG file.


REVTRACK_USER

Username used to log on Windows will be recorded into Permanent.log file (if revision tracking enabled). If we want to record something else, we have to create this symbol with desired string value.


SENDTO

Default e-mail address.


SHOWDSN

If this symbol is defined and has the value 1, current datasource name will be displayed in the program caption (title).


SORTWORK

If Iaf programs (reports or batch) process files (indexed or sequential), Iaf uses an internal aglorithm to sort records of main input file (if necessary). During sort phase, temporary files will be created in working directory. At the end of the program, these temporary files will be
deleted. Using this symbol, we can declare a directory in which temporary files will be created.
