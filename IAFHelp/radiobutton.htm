Radio button
   

We can consider radio button as a destination field which stores one of many values: 0 or 1 or 2 or 3, "A" or "B" or "C" or ...

To insert new radio button on the form, click button Radio button in toolbox or choose menu command Objects/Radio button. Move the cursor to the desired position,  then click.
Definition dialog appears on the screen.

 

One radio button stores one value into the destination field. Because of this, we always create radio buttons in a group, where each radio button stores different value into the destination field. The group of buttons is defined implicitly: all buttons inside a frame represent one group. This means that the frame around the group is required at paint-time. If we do not want to see the frame at run-time, we simply set the Visible attribute to No.
Initially, maximum 10 radio buttons can be defined. Later on, if we want to add new radio button in the group, we can use Copy/Paste operation. All radio buttons in a group have successive sequence numbers. The object with smallest sequence number is the first element in the group.

  
Frame text
Frame label.
Variable
Destination variable.
Value
This value will be moved into destination variable when radio button is checked.
Text
Radio button label.
Default
Default value will be selected when runtime displays the form.
Orientation
Vertical (one button below another) or horizontal (one button next to another)
Read only
If radio button is read only, mouse click and keyboard input is not allowed.

The number of value/text pairs defines the number of radio buttons in the group.

New group of radio buttons is placed on the form: the frame and radio buttons are displayed on the screen, and one line is placed to the code buffer of each radio button. The line is

Destination = variable_name/checked:V1

This statement means the following: if radio button is clicked, move V1 to variable_name.

The code buffer for the first radio button (Option 1) contains

Destination = $opt /checked: "A" /defval: "A"

The code for the second radio button (Option 2) is

Destination = $opt /checked: "B"

The code for the third radio button (Option 3) is

Destination = $opt /checked: "C"

The radio button object has the two components: visual appearance (radio button and text label) and code buffer (contains statements defining the output variable, "checked" value, additional actions after click, etc.).

To change radio button properties double click on the radio button

-or-

select one of radio buttons, right click to display context menu and choose Properties.

How run-time engine processes the radio button group?

If one object is checked, run-time stores "value" to the destination field. Only one radio button in the group can be checked.
