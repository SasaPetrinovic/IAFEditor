Check Box
   

We can consider check box as a destination field which stores one of two values: 0 or 1, "Y" or "N", etc.

To insert new checkbox on the form, click button Checkox in toolbox or choose menu command Objects/Checkbox. Move the cursor to the desired position,  then click.
Definition dialog appears on the screen.


  
Caption
Check box label
Variable
Destination variable. Run-time engine will move one of two values into this variable.
Unchecked
This value will be moved into destination variable if check box is not checked.
Checked
This value will be moved into destination variable if check box is checked.
Default
Default value will be selected when runtime displays the form.
Read only
If check box is read only, mouse click and keyboard input is not allowed.


New checkbox is placed on the form: check box button and text label are displayed on the screen, and one line is placed to the code buffer. The line is

Destination = variable_name/checked:V1/unchecked:V2

See Destination statement.


The check box object has the two components: visual appearance (check box button and text label) and code buffer (contains statements defining the output variable, "checked" value, "unchecked" value, additional actions after click, etc.).

To change checkbox properties double click on the checkbox

-or-

select the checkbox, right click to display context menu and choose Properties.


How run-time engine processes the check box?

If object is checked, run-time stores "checked value" to the destination field.


If object is not checked, run-time stores "unchecked value" to the destination field.
