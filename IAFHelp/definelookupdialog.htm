Define lookup dialog
   

Lookup dialog displays rows from a query or elements of arrays. By clicking 'Move' button, one value can be moved to the main program.

The LOOKUP dialog can be used in a program, together with a DESTINATIONFIELD, via /LOOKUP qualifier. Objects in the dialog are: list control and 2 buttons (Move and Cancel). Our task is to define dialog title and list control columns.

Title

Dialog title, an alpha literal or a local field. At run-time, this text is displayed in dialog caption.

Select command (query)

SQL SELECT commands defines input record stream. The fields from select_list will be displayed in the list control, therefore, number of fields determines number of columns in the list control.

There are two ways to enter SELECT command: manually and automatically.


·     Manually

Click the 'SELECT command' edit box and enter your statement in the form

SELECT fld1, fld2, ... fldN FROM mytable  [WHERE ... ORDER BY ...]

When finished, click 'Parse SELECT command' button. If the select list is correct, appropriate number of columns will be created in the list control which is placed on the right side of the dialog. By clicking columns header, you can customize all the columns (column header, alignment and edit mask).


·     Automatically

Click the 'Insert SELECT command' button. In new dialog select table and its fields.


Hints


·     Tables used in the query don't have to be declared in Datasource dialog.


·     If we use syntax

            SELECT fld1 INTO :hostvar1, fld2 INTO :hostvar2 FROM tblx

            table TBLX doesn't have to be loaded into the DD (all fields from the select_list are stored into host variables).


·     If we use derived table in the query, such as

            SELECT fld1, ..., fldN FROM (SELECT ... ) as t1 ...

each field from the select_list must be stored into a host variable, ie.

            SELECT fld1 into :hv1, ..., fldN into :hvN FROM (SELECT ... ) as t1 ...


·     When we use a function in the select_list, INTO keyword is required:

            SELECT fld1, sum(fld2) into :hv FROM tbl GROUP BY fld1


·     Parameters are denoted by colon sign (:)

            SELECT fld1, ..., fldN FROM tbl WHERE fld1=:hostvar


·     Implicit joins can be used:

            SELECT fld1, ..., fldN FROM tbl1, tbl2 WHERE tbl1.pk=tbl2.pk


·     Explicit joins can be used as well:

            SELECT fld1, ..., fldN FROM tbl1 LEFT JOIN tbl2 ON ...


·     Comment lines can be used before SELECT statement

            ! comment, line 1
            ! comment, line 2
            ! comment, line 3
            !
            SELECT  fld1, ..., fldN FROM tbl1


Note 1

            When we use "Select fld INTO :hostvar" clause, host variable hostvar must be declared in all programs that use the lookup dialog. To isolate lookup dialog from other parts of the program, several predefined variables have been defined:
  
LOOKUP_Ann
ANSI alpha field, length = nn characters
LOOKUP_Wnn
UTF-16 alpha field, length = nn characters
LOOKUP_DT
Datetime
LOOKUP_D
Date
LOOKUP_Ndd
Numeric field with dd decimals

            Examples:

            SELECT fld1, sum(fld2) into :LOOKUP_N2 FROM tbl GROUP BY fld1
            SELECT Left(fld1,20) into :LOOKUP_A20 FROM tbl

Note 2

Instead of a SELECT statement, arrays can be used in the dialog. This means that elements of vectors will be displayed in the dialog. The format is:

ARRAYS arr1(), arr2(), ..., arrN() NUMBER_OF_ROWS numeric_variable

Arrays $v1 and $v2 must be filled before the field with /LOOKUP qualifier.

Examples:

ARRAYS $v1(), $v2() NUMBER_OF_ROWS 10
ARRAYS $v1(), $v2() NUMBER_OF_ROWS counter_2

$v1(1) and $v2(1) will be displayed in the first line, $v1(2) and $v2(2) in the second line, etc.

Data entry field

Lookup dialog enables defining one data entry field named LOOKUP_FIELD. This field can be used in SELECT statement as a parameter in the form:

SELECT ... FROM table WHERE ... a_field like :LOOKUP_FIELD ...

Field prompt - text to be displayed as a label.
Field type - alpha ANSI or Unicode.
Field size - allowed number of characters.

At run-time, lookup dialog appears on the screen, waiting for data entering. After data entry, SELECT statement executes and displays result in the list.

Example:

Select fld1, fld2 from tbl Where fld1 Like :LOOKUP_FIELD

List control

Lookup list has row heading: check this option if you want to display ordinal number of the line in the first column.
Lookup list has grid lines: check this option if you want to display gridlines around lines.

To customize particular column of the list control, click column header.

Heading text     - can be alpha literal or a local field.
Alignment         - left, center or right.
Edit mask         - see Edit mask.
Move action      - set check box if the value from that column should be moved to the main program.

Connection string

Connection string is alpha literal or local field whose form is:

dsn_name;UID=username1;PWD=password1

Examples:


·     'dsn1;UID=user1;PWD=pwd1'
·     $mydsnstring. This variable should be filled anywhere in the program, before activating LOOKUP dialog.


Description

Short description of a dialog, plain text. Used for documentation purposes only.
