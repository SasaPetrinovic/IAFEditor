Button properties
   

Object name

When the new button is created, generator automatically assigns the name Button_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Member of group (information only, cannot be changed)

If the object is a member of a group, the name of this group is displayed. An object can be the member of one group only.

Caption

The text which appears on the button. If you want to define one letter as a prefix (mnemonic) character, precede it with ampersand (&). At run-time, that letter will be underlined and user can click that button by pressing Alt + letter.
If you want to display single ampersand in the caption, enter two consecutive ampersands.

Within one subform, all prefix characters must be different.

To change caption text dynamically, use the Display method.

Sequence number

If the button has sequence number 0, click is allowed at all times if the button is enabled.

If the button has nonzero sequence, click will be accepted only if all objects with the smaller sequence number are processed.

Visible

Determines whether the button is visible when the subform is first displayed.

Enabled

Determines whether the button is displayed as disabled when the subform is first displayed.
When the button is disabled, mouse clicks are ignored.

Font name

Choose one from offered font names.

Font size

Specifies font size of the text.

Bold

Specify that font weight is 700. Normal weight is 400.

Italic

Specify whether text is italic.

Underline

Specify whether text is underlined.

Tooltip text

Tooltip text is a text that will be displayed when the cursor hovers above the button. See Tooltip text definition.


Button style

Values:
  
Standard, no shading defined
Standard push button. If the themes are active, this option will render a standard push button.
Standard, shading defined
The themes will be ignored and internal drawing function will be used to render the button.
Flat
Flat form, not three-dimensional.
Hyperlink
Hyperlink form. When the cursor hovers above the text, it changes its shape into the pointing hand. The Caption specifies an arbitrary text.
Hyperlink, variable caption
Hyperlink form. In this case, Caption specifies variable name. At run -time, contents of this variable will be displayed as a hyperlink text. This option is especially interesting if the button has been used in a grid. For example, if the caption attribute contains text $TEXT, value of the local field $TEXT will be used as a hyperlink text. Also, a field from a table can be specified.

The same result can be achieved, if we choose simple hyperlink
attribute, and at run-time display variable text using the Display
method, for instance, Table_1.Display(current_line,5,$text).

It is much better choose the first option (variable caption).
Flat with shading
This option combines flat and shading attributes.

Cancel button

If this attribute is checked, when click occurs, the program immediately leaves current field without further actions. In other words, no field calculations, no field checking, no line ending actions (if we process a grid). All cancel buttons have sequence = 0. See also 'Causes validation' attribute.

Submit button

If this attribute is set, additional checks are performed before activating click actions: all destination fields in current subform must already be processed with no errors. If check algorithm discovers an invalid field, input focus will be set to that field waiting for data correction.

Notes
            - If the button has sequence = 0, program does not automatically leave the subform. Programmer has to put FINISH_WINDOW command in click actions.
            - If sequence <> 0, the button has to be the last object in the subform (the largest sequence number in the subform).

Multiline

Word wrap is on. If the caption is too long to fit in a single line, text will be displayed in multiple lines.

Causes validation

This attribute is applied only if a destination field has the focus during the click.

When the attribute is cleared, there is no control of the active field, but line ending actions will be performed if we enter a grid field. Click actions of the button start immediately. This means that this attribute needs to be cleared in Cancel or Exit buttons.

When the attribute is set, then the value of the active field will be checked.
First, the correctness of the entered value (date, time, alpha) checks. After that, screen value moves into the field buffer and external control starts (instructions from DESTINATION up to the end of the block). If everything is all right, click is accepted and click actions activated.

This attribute is usually set in Ok, Print or similar buttons.

What is the difference between Cancel button and No validation attributes?

If 'No validation' attribute was set, program performs Line ending actions if we enter a grid field, and after that activates click actions.
If 'Cancel button' was set, click actions will be activated immediately.

Menu

Attaches popup menu to the button. If you select this attribute, context menu for this object contains additional two choices: Define menu layout and Define menu code.

Menu position

Position of the menu in regard to the button, down or right.

Text color

Color of the text.

Active text color

This attribute is applied if button style is Flat. Specifies text color when cursor is above the button.

Active background color

This attribute is applied if button style is Flat. Specifies background color when cursor is above the button.

Shading style

Specifies that Color_from fades gradually to Color_to.

Values:

None                No shading
Horizontal
Vertical

Shading variant

Valid if shading style is Horizontal or Vertical.
Possible values are 1 to 4.

Color from

Starting color. Valid if shading style is Horizontal or Vertical.

Color to

Ending color. Valid if shading style is Horizontal or Vertical.
