READ_EXTERNAL
   

This statement reads symbol.

Format 1

            READ_EXTERNAL /NAME: symbol_name     field1 [, ..., fieldN]

Format 2

            READ_EXTERNAL /VARNAME: alpha_variable     field1 [, ..., fieldN]

Where


·    symbol_name - the name of a symbol. The first character in the name must not be underline (_). If the first character is a dollar sign ($), symbol will be read from the system registry. Otherwise, symbol will be read from symntt00.iaf file. If the symbol name contains spaces, an apostrophe or quotation mark has to be used as delimiter.
·    alpha_variable - variable containing symbol name.
·    field1, ..., fieldN - list of output fields: the fields where read value will be stored.


Qualifiers

            /NAME defines symbol name.
            /VARNAME defines an alpha variable containing symbol name.

Notes


·           Registry key is HKEY_CURRENT_USER\Software\Sing\IAFW\RT\symbol_name.

·           Also, you can read any symbol from HKEY_CURRENT_USER branch. Symbol name has to be in the form HKEY_CURRENT_USER\sub_key1\sub_key2\...\symbol_name.

·           If symbol does not exist, output fields will be initialized.


Examples

  
! Read SYM1 from Symntt00.iaf
    READ_EXTERNAL/NAME: sym1 counter_1
! Read from Registry, key=hkey_current_user\software\sing\iafw\rt\sym1
    READ_EXTERNAL/NAME: $sym1 counter_1
! Read from Registry, key=hkey_current_user\software\sing\iafw\rt\sym2
    READ_EXTERNAL/NAME: $sym2 counter_1,counter_2
!
        READ_EXTERNAL/NAME: $hkey_current_user\subkey1\sym1 counter_1
! Symbol name contains space
        READ_EXTERNAL/NAME: '$hkey_current_user\sub1 abc\sym1' counter_1
        READ_EXTERNAL/NAME: "$hkey_current_user\sub1 abc\sym1" counter_1
! Using /varname
    $sname = "$hkey_current_user\sub1 abc\sym1"
        READ_EXTERNAL/VARNAME: $SNAME counter_1
