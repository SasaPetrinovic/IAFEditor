Additional view actions
   

As we already seen, the view determines selection criteria, joins, sort order etc. In many cases, this is quite adequate to define main input record stream. But, if we have some complex selection criterion we can calculate it at this place and, eventually, ignore the record if the criterion was not satisfied. To ignore current record, we use NEXT statement.

Additional view actions execute just after the reading of record from input record stream.

Example:

            calc1
            calc2
            ...
            if $flag = 0
                        NEXT
            end_if

This means the following: if $FLAG has value 0, then read next record from the input stream (the current record will be ignored).
