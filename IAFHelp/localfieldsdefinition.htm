Local fields definition
   

Local fields are variables defined in a program and can be used in that program only. These variables can be seen from any part of the program (field calculations, subform calculations, local procedures, global procedures, ...). Local fields are usually used as work variables, to save temporary  results, etc. Maximum number of local fields in one program is 500.

Each editor line defines one local field. The line format is:

local_field_name   field_type   size   [data_type]  [occurence]   [!!comment]

local_field_name = local field name is maximum 60 characters long, the first character must be a dollar sign ($).

field_type = field type mnemonic.

  
field_type mnemonic
field type
 
 
A
alpha ANSI: 1 character occupies 1 byte, max length 32767
N
numeric
D
date
T
time
W
alpha, UTF-16: 1 character occupies 2 bytes, max length 32767
DT
datetime
S
memo ANSI
SW
memo unicode utf-16

memo: variable length alpha field.

size = field length. For numeric fields this parameter is in the form tot_digits[,dec_digits].

data_type = applies to numeric fields only.

  
data_type mnemonic
data type
 
 
U
unpacked numeric (default)
P
BCD packed numeric (obsolete)
W
integer, 2 bytes (word)
L
integer, 4 bytes (longword)
Q
integer, 8 bytes (quadword)
D
double floating, 8 bytes

occurence  - number of array elements, if we want to define an array. Max value 65535.
