Show object list
   

This dialog lists all the objects in the subform or report. Some suspicious objects (width and height equal 0; x,y beyond subform limits, invalid field type, etc.) are denoted by a question mark. In addition to list review, you can perform some housekeeping functions (remove an object, reset coordinates, hide object, sort z-order, etc.)

Delete

Removes selected object from the subform / report.

Reset X,Y

Sets x = 0 and y = 0.

Reset gradient

Resets gradient information (shading style, shading variant, gradient colors).

Hide

Temporary hides selected object. This option is very useful if two or more objects overlap.

Unhide

Unhides selected object.

Unhide all

Unhides all hidden objects.

Set to Alpha, Num, Date, Time

When a sourcefield or destinationfield object is created for the first time, the attribute "field type" is set to one of values "Alpha", "Num", "Date" or "Time". For example, Source=acc_1 has field_type=Num because acc_1 is a numeric field. Later, using editor we can change this instruction to, for instance, Source = program_name. Edit mask can also be changed from "ZZ9,99" to "XXXXXXXXX"; see dialog "Change edit mask". The problem is that painter does not change the field type attribute automatically. Now, the situation is next: sourcefield object displays the alpha field (program_name), but the field type attribute remained "Num". This difference could bring us some difficulties at run-time. To avoid such problems, we can manually set the field type attribute.

Sort Z-Order

This button is visible if some objects have irregular position in the list of the objects.
The regular order is:
·     subform (if exists)
·     all passive objects in ascending Z-Order
·     all active objects with ascending sequence number.
