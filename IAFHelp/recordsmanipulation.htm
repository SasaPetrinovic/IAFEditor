Records manipulation
   

The list shows already defined records. One line contains the following information: record name, number of fields, record size (in bytes), number of keys and date of last modification.

Record name is maximum 30 characters long, the first character must be a letter. Allowed characters: English letters A - Z, digits 0 - 9, underline (_) and a dollar sign ($). Record name in one Data Dictionary must be unique.

Modify

To modify an existing record definition, select the record in the list and click Modify

-or-

Double click the record name in the list.

The record definition dialog appears on the screen.


New

To create new record definition, click button New and go through the record definition dialog.

Copy


To copy a record definition to another record, select the record in the list and click Copy. This option is very useful if we have similar record definitions.


Export description

Export record definition to a text file (default file extension is RDS). Such text file could be copied to another PC.

Import description

Import record definition from a text file previously created by Export description.

Analyze data

If you have a text file (*.txt, *.csv, etc.), possible export from a database or something else, you can analyze it, and create a record definition based on that analysis.

Delete


To delete a record definition from the Data Dictionary, select the record in the list and click Delete.

Delete unused records

To delete all unused records (the records which do not appear in any file definition), click Delete unused record. The list of unused records appears on the screen. We can delete all these records or just some of them.

Unlock

If during record modification phase Data Dictionary Maintenance program crashes, the record remains locked. To unlock the record, select it and click Unlock.
