Table modification
   

This dialog enables you to change table structure just in your Data Dictionary. Table structure in database will not be modified.

All modifications are in effect until next Load table definitions from datasource.

In the most number of cases, you should not modify table structure. But, there are situations where the modification is necessary.

Some fields could be simply removed from the table definition. As we already said, table structure in the database remains unchanged, but our programs will not deal with these removed field. Primary key fields must not be removed.

Some ODBC drivers do not return information about primary key of a table. This information is very important to the generator, and you should specify primary key segments manually (grid column Key).

Also, some ODBC drivers, like driver for DB2, do not return proper autoincrement information of a column. Autoincrement columns must not be used in SQL INSERT and UPDATE statements. This is a reason you should adjust this attribute manually, if the loading process have not done this properly.

Field name can not be modified because it represents field name used in I/O statements (SELECT, INSERT, UPDATE, ...).

Global field name is template field created automatically during loading phase and should not be modified without a serious reason.

Occurrences enables you to emulate an array.
Suppose we have a field in a table defined as: FLD1 varchar(100). After loading into the DD, we will get: Fieldname = FLD1, Globalfieldname = ODBC$VAR0100. All the programs will see this field as an ordinary alpha field, size = 100 bytes.
But, if we want to consider this field as an array of 25 elements, each element 4 bytes long, we can do the following: define a global field AL04  (alpha, size = 4 bytes), change ODBC$VAR0100 by AL04 and enter 25 into the "occurrence" column. After that, in program generator, we can use this field as an indexed one: FLD1($ndx) = "ABCD" where $ndx is between 1 and 25.

Each field has local attributes (unlike global attributes defined by global field):

·     Column header (used in default programs in Form or Report painter).
·     Help text, maximum 80 characters, used in data entry programs.
·     Default value, a literal. This value will appear on the screen at run-time when an user try to enter value from the keyboard into this field. We can override this value by /DEFAULT qualifier in Form painter, see the Destination statement.
·     Range control in the form minimum ... maximum. This control executes after field entering. Examples: 'AAA' ... 'GGG' or ... 'KKK' or 'GGG' ...
·     Description, maximum 80 characters.


Detail characteristics of global fields we can see by clicking Show Fields.


How to remove a field?

Click side heading of the grid. Whole line becomes selected.
Press "Delete" key.

How to change field position?

Click side heading and drag the field name down or up to the desired position.


Save & Exit

By clicking this button, new table structure will be saved in Data Dictionary and text file tablename.clb will be created. This file contains record definition for C compiler.
