TRNLOG
   

This function translates a logical name.

Format

            TRNLOG(logical_name)

Return Value Type

            Alpha.

Parameters


·     logical_name - alpha literal or alpha variable containing logical name.


Note

            All definitions of logical names are stored in the file logntt00.iaf, see Logical Name Table Maintenance.

            The question is: how a program knows where the file logntt00.iaf resides on? The algorithm is the following: Run-time engine retrieves full path of the executable program. If the path does not start with two backslashes (\\), run-time engine tries to open file on this directory. If the path starts with two backslashes, denoting a local area network location (according to Universal naming convention, file name is in the form \\servername\sharename\path\file_name), run-time engine retrieves current working directory for the current process and try to open file from that location.
            In the other words, if programs reside on local PC, logntt00.iaf must be at the same directory as executable programs. If programs reside on server PC, runtime engine looks for logntt00.iaf at working directory.


            Working directory can be set in two ways:
·           via menu interpreter command (IAFMENUN menu_name working_directory) or,
·           using command line parameter: program.exe WORK=work_dir.


            Result is an alpha field consisting physical path.

Examples

  
      $res = Trnlog("apli_exe")
      $res = Trnlog("apli_dat")
