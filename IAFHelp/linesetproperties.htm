Lineset properties
   
Print page header on the first page only?

Valid for page header only. If you want to print page header only once at the beginning of the program, select this check box.

Print on page overflow?

Valid for sub-headers only. If you want the sub-header to be printed always when the skip to the new page occurs, select this check box. This option is recommended in order to reach better report layout.

Next lineset overlaps this one?

Valid for sub-headers only. Generally, after printing of a sub-header, print position (vertical offset from the top of the page) increases by sub-header height. Next lineset starts from new print position. If this options is checked, next lineset starts from the same print position as this subheader.

Is this lineset repetitive?

Generally, one input record generates one detail line, or activates break which causes printing one group lineset. In some cases, we need to repeat a lineset many times with various values.

Example: a customer takes loans from a bank in various currencies. In detail line we can print every loan (amount, currency, dates, etc), but in subtotal line we want to print total amounts per each currency. The simplest way is to fill two arrays in detail line (array_1 for currency names, array_2 for sums) and a variable holding number of different currencies. In subtotal line we need to draw two source fields: $array_1(repetition_line) and $array_2(repetition_line), where repetition_line is internal variable denoting current repetition (1, 2, ...).

Repetition counter

If the lineset is repetitive, this counter (a numeric integer variable), defines number of repetitions.

Background style

If you want to define lineset color, choose NORMAL.

Background color

Valid if background style set to NORMAL.

'Zebra' style

Valid for detail and the first subtotal, ie. for linesets which repeat one after another.
If this check box is selected, one line will have white background, the next one will take background color, and so on.

Print vertical gridlines

If you want a tabular-like report with vertical lines, select this check box.

Print horizontal gridlines

If you want a tabular-like report with horizontal lines, select this check box.

Line start (in pixels)

Valid if 'Print vertical gridlines' = No and 'Print horizontal lines' = Yes. Start of the horizontal line.

Line end (in pixels)

Valid if 'Print vertical gridlines' = No and 'Print horizontal lines' = Yes. The end of the horizontal line.

Line style

Valid if you use gridlines.

Line width

Valid if you use gridlines. Line width 0 means 1 pixel width.

Line color

Valid if you use gridlines.

Vertical gridlines positions

If you use vertical gridlines, you have to write down positions of vertical grid lines.

Format is:

pos1, pos2, ... posN

where pos1, pos2, ..., posN are integers separated by commas. Values should be stated in ascending order.

The horizontal line starts at pos1 and ends at posN.

Example

1, 70, 130, 628

which means: there exists four vertical gridlines with positions 1, 70, 130 and 628 pixels. There is one horizontal line from 1 to 628.


Note

For page header we can define background style and background color only.
