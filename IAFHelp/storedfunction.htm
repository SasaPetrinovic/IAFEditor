STORED_FUNCTION
   

This function calls a stored function via ODBC. Stored function is a saved SQL routine that returns a value. Iaf can call stored functions that return scalar data types: number, date, character.

Format

            STORED_FUNCTION(function_name[,arg1][,arg2][,…argN])

Return Value Type

            Alpha, numeric or date.

Parameters

·     function_name - alpha literal, name of the stored function. Function name can be qualified by owner, database or catalog. Portable delimiter is DOT. If program uses Informix, this delimiter will be replaced by COLON. Valid names are "fnc1", "owner.fnc1", "database.owner.fnc1" (for MS SQL Server).
·     arg1,..., argN - arguments of the stored function. An argument can be: NULL (null value), literal or variable.
Notes

            The function can be used in an assignment only. Syntax is:

            variable = STORED_FUNCTION(function_name [,arg1]...[,argN])

            If  the result can be NULL, the left-hand variable must be one from a database table,             because standard Iaf variables have no NULL flag.

Example
  
      counter_1 = STORED_FUNCTION("fnc_noarg")
      counter_1 = STORED_FUNCTION("fnc1", NULL, "ABC", $FLD1)
            tblfld = stored_function("fnc1",arg1)
      if tblfld NULL
            ...
      end_if
!! we cannot use: if counter_1 NULL
