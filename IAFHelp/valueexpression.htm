Value Expression
   

Definition 1

A value expression (value_exp) is a symbol or string of symbols used to calculate a value.

Definition 2

A value expression (value_exp) is a combination of variables, operators, constants and literals that calculates a value.


The value can be:
1.   alpha value (alpha_exp). Alpha field is ANSI if each character occupies one byte, or Unicode (UTF-16) if one character occupies two bytes.
2.   numeric (numeric_exp)
3.   date (date_exp)
4.   time (time_exp)

Literals

An alpha literal is zero or more characters enclosed in delimiters: double (“) quotation marks or apostrophes (‘). First and last delimiter must be the same.

To include quotation marks in a literal, use apostrophes as delimiters and vice versa.
Any non-printable character can be entered into alpha literal using hexadecimal escape sequence in the form <xNN>. Such sequence contains exactly 5 characters: <, little x, two hexadecimal digits denoting a character and >. For example <x0A> denotes line feed, <x0D> carriage return, <0x1B> an escape character. To enter an Unicode character, 7-character sequence <wNNNN> should be used. For instance <w0401>, <w0110>.

If you want to enter a cyrillic alpha literal in the text editor on PC with ansi code page different from 1251, you should use the form C'latin_literal' or C"latin_literal"; for example  C"ABCD".

Examples
  
            “ABCD"
      'A1234'
      "ABCD<x0d><x0a>" !! ABCD + CRLF
      'ABCD'
      'ABC"D'
      'ABCD" !! invalid literal, different delimiters
            C"abcd"                        !! result is "????"
            C'ABCD'
 

A numeric literal is a string of digits with an optional sign (+ or –) and decimal point.
Decimal point is defined in Change/define setup parameters dialog. Two values are allowed:
comma (,) or point(.).

Examples
  
!!    a/ decimal point is COMMA:

            100,00
            -12
            +134
            -1,12

!!    b/ decimal point is POINT:

            100.00
            -1.12
 

A date literal has the form “[D]D-MMM-YYYY” where is
            [D]D – one or two-digit day
            MMM – three-letter month (in English: jan,feb,mar,apr,may,jun,jul,aug,sep,oct,nov,dec)
            YYYY – year

The alternative form of date literal is "YYYY-MM-DD[ HH:MM:SS]".

Examples
  
      "1-jan-2002"
      "01-jan-2002"
      "01-JAN-2002"
      "30-MAY-2002"
      "2008-07-01 10:00:00"
      "2009-01-05"
 

A time literal has the form “[H]H:[M]M[:SS]” where is
            [H]H – one or two-digit hours (0 to 23),
            [M]M – one or two-digit minutes (0 to 59)
            [S]S – one or two-digit seconds (0 to 59)

Examples
  
      "09:30"
      "9:15"
      "9:5"
      "20:5:20"
 


Constants

Constants are predefined variables whose value cannot be changed.
According to this definition, constants cannot be used in an assignment on the left-hand side of the statement.

See Constants.

Local fields


Local fields are defined in the particular program or the procedure (function).

In the program, name of local field must start with $ (dollar sign) and maximal name length is 16 characters.
In a procedure (function),  name of local field must start with $$ (two dollar signs).

Local fields in a program are defined in Define local fields dialog.
Local fields in a procedure (function) are defined using DEFINE_LOCAL instruction.

Format:

            $localname                                definition in a program
            $$localname                              definition in a procedure (function)

Local fields defined in a procedure (function) are visible in that procedure (function) only.

Local fields defined in a program are visible in all segments of the program and in all procedures (functions) used by the program.

The maximal number of local fields in the program is 400.
The maximal number of local fields in the procedure (function) is 100.


Examples

  
      $locnam1
      $loc1
      $longlocname12345678    !! invalid name, length > 16
 



Database fields


A database field is the field from a table (record) which has been described in Data Dictionary.


Format

            [tablename.[recordname.]] fieldname [(index)] [[start:length]]

            Index is an integer greater than zero, used for access to elements of arrays. The first element has index 1, the second 2, etc.
            Start and length describe a part of an alphanumeric field. Start is an integer greater than zero and specifies the position of partial field. The first character has the position 1, etc. Length specifies the number of characters in the partial field.

Note 1

If in a program two or more records use the same field name, the generator will take the field from the first file in which appears this name. The order of files is defined in 'Datasource dialog'. In order to avoid possible errors, table qualifier need to be used (table.field or table.record.field). We use record qualifier if a file has more record redefinitions.

Note 2

Most of databases do not allow arrays in table definition. Arrays can be defined in indexed or sequential files. Therefore, database field have the format

            tablename.fieldname[start:length]

Note 3

Record redefinition is notion which exists only with files (indexed or sequential). This means that in one file can exist records with different structures. Thus, file.record.field syntax can be used in programs working with such files.


Examples

  
      field1            !! ordinary field
      field1(3)         !! indexed field
      field1(counter_1) !! variable index
      field1[1:5]       !! partial field
      field1[counter_1:5]     !! variable partial field
      field1[counter_1:counter_2]
      field1(3)[3:5]    !! indexed and partial field
      file1.field1      !! using file qualifier
      file1.rec1.field1 !! file, record qualifier
