Barcode printing
   

Barcode can be printed on laser or inkjet printers in the two ways: using a barcode font or via internal routines.

Barcode font

If you have a barcode font installed, just create sourcefield object, select Source field style = standard (this is the default) and choose barcode font (in 'Property' dialog).

A little disadvantage is that the font must be installed on every PC running the program.


Internal routines

To use this feature, create sourcefield object and select Source field style = Barcode_XXX.

The generator can print the following barcodes: EAN13, EAN8, Code 39, Code 93 and Code 128.

EAN13 is numeric barcode used to identify product at the point of sales.
Valid characters: 0123456789
Field length is fixed, 13 digits: 12 data digits + check_digit (modulo 10).
If program variable contains 12 digits, the generator automatically calculates and adds check digit.

EAN8 is short version of EAN13 for small products. This code can be used only if EAN13 occupies over 25% of the front sufrace of the product.
Valid characters: 0123456789
Field length is fixed, 8 digits: 7 data digits + check_digit (modulo 10).
If program variable contains 7 digits, the generator automatically calculates and adds check digit.

Code 39 is an alphanumeric barcode used in internal applications. The density is not so good.
Valid characters: letters A-Z (upper-case only) , digits 0-9 and space, minus(-), plus(+), period (.), dollar ding($), slash (/) and percent (%).
Field length is variable, no check digit.

Code 93 is more compacted version of Code 39 and occupies significantly less space.
Valid characters are the same as with Code 39.

Code 128 is modern alphanumeric barcode with high density. The density is excellent for numeric fields.
Valid characters: all ASCII 7-bit characters (from 0 to 127).
Field length is variable, the generator automatically calculates and adds check character (modulo 103).

Dimensions: to print barcode with precisely defined dimensions, choose report type Label or Form and check the option 'Draw in physical units'. In standard reports (report type=report), barcode dimensions will be approximately precise.

During barcode printing, the generator considers the following attributes: orientation (0, 90, -90), text color, background style, background color, font.


Note

There are many specialized barcode label printers. The generator can print barcode on these printers, too. In that case, a text file with data and appropriate ESC sequences need to be created by a program. After that, we can copy this file to the output device.
