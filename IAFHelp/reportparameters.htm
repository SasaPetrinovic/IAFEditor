Report parameters
   

Paper dimensions

            Page width
            Page height
            Page overflow is the position where the skip to new page occurs.

            Based on these dimensions (width, height) report painter creates work area: 1 mm = 3 pixels.

Orientation

            Portrait
            Landscape

            This parameter defines work area appearance.

Drawing area

            Default is 1 page. If a report has many subheaders and subtotals which cannot be fitted into one page, increase working area to two pages.

Default orientation at run-time

            Portrait
            Landscape
            
            This parameter defines default orientation in Print dialog at run-time.

Orientation can change at run-time

            If this option is not checked, the paper orientation cannot be changed at run-time.

Draw in physical units (mm) at run-time

            This option is available for the form report type only.

Number of copies

            Numeric literal or numeric variable. If blank, 1 is assumed. This parameters is valid only if we choose direct printing in Print dialog (click OK).

Form name

            This parameter is mainly used for nonstandard paper size. Run-time engine try to sets paper size to "form name" if the default printer supports such paper. If this parameter is blank, default paper will be determined according to report dimensions (page width, page height).



Print-file name

            If we click Preview button in Print dialog at run-time, report will be written into the print file. Contents of the print-file can be reviewed by Iafpreview utility program. Print-file name can be 30 characters long.

            Print-file name is:


·           Standard file name, like myrep.prt. This print file will be created at work directory. When we activate the program next time, new print file will overwrite the old file. Absolute path can be used, too: c:\temp\myrep.prt.


·           File name with logical name, like myprint_dir:myrep.prt. Logical name myprint_dir must be defined in Logntt00.iaf.


·           Versions: if file name contains # sign, run-time engine will create various versions of the file. Number of # signs determines number of versions: using one #, we can create 9 versions, two #'s enables 99 versions, etc. For example, by using myrep#.prt, nine files can be created: myrep1.prt, myrep2.prt, ..., myrep9.prt. File name myrep##.prt creates myrep01.prt, myrep02.prt, ..., myrep99.prt. After myrep99.prt, myrep01.prt will be created.


·           Variable print-file name: if we enter a local variable as print-file name, then we can set its value programmatically. The variable have to be filled before Print dialog appears on the screen. For example, if we enter $printfile as print-file name, then somewhere in program title we must set up this variable:
            $printfile = "myrep.prt"   or
            $printfile = "repdir:myrep.prt"      or
            $printfile = "myrep##.prt"


Document parameters

The following parameters will be written into the print file. The print file will be created if we click Preview button in Print dialog or if we set IAF_PRINTMODE internal variable to 9 or 12 (See Internal fields).


Title

This text will be displayed as a caption when we review print file. Enter alpha literal or alpha variable (ANSI or UTF-16). If we use alpha variable, its value should be set before the end of program.

Examples:
            "This is a title"   - alpha literal
            $title                 - local variable

Subject

Short program description (either alpha literal or alpha variable (ANSI or UTF-16)).

Example:
            "This is statement of account"

Author

Author of the program (alpha literal or alpha variable (ANSI or UTF-16)). Default value is name of the user currently logged on the system.

Open action

The action when the print file opens for the first time:
·     Display report header (if exists),
·     Display recapitulation (if exists), or
·     Display any page

Show mode


·     Display page only.
·     Display page and bookmarks tree.
