Some of these fields are read-write: can be changed and used as work variables (ACC_nn, COUNTER_nn, EOF, IAF_PRINTMODE, NUMBER_LABELS, PAGE_COUNTER, STARTING_LABEL).
The other fields are read-only.

For detailed explanation of these internal fields see below:


·     ACC_nn
  
Accumulators are signed numeric 18-digit fields. Number of accumulators is 40.
            
Format is the following:

ACC_1  to ACC_20  have 2 decimals (integer part=16 digits, decimal part=2 digits)
ACC_21 to ACC_30  have 0 decimals (integer part=18 digits)
ACC_31 to ACC_35 have 3 decimals (integer part=15 digits, decimal part=3 digits)
ACC_36 to ACC_40 have 8 decimals (integer part=10 digits, decimal part=8 digits)

Accumulators can be used as working fields, to save temporary result, etc.

·     CMD_PARAM

  
CMD_PARAM contains value of named command line parameter PARAM. This is alphanumeric field, maximum length is 255 characters.

Example:

If we activate an Iaf program using command line, like

            Myprog.exe PARAM=par1

then later in the program we can test internal variable CMD_PARAM

if CMD_PARAM = "par1"
      ...
end_if

If parameter contain blanks, quotation marks are required:

Myprog.exe PARAM="xx yy zz"

·     COUNTER_nn

  
Counters are 6-digit numeric fields without decimals. Number of counters is 20, which means that nn is in the range 1 to 20.


·     CURRENT_COLUMN

  
Used with grid (table) object only. Specifies the column in which the cursor resides (data-entry grid), or selected column (display-only) grid. This is numeric field, minimum value is 1.

Example:

Table_1.settextcolor(current_line,current_column,"red")


·     CURRENT_LINE

  
Used with grid (table) object only. Specifies the line in which the cursor resides (data-entry grid), or selected line (display-only) grid. This is numeric field, minimum value is 1.

Example:

Table_1.settextcolor(current_line,1,"red")


·     CURRENT_MOVE_COL

  
Used with grid (table) object only. Specifies the column over which the cursor moves. This is numeric field, minimum value is 1. Usually used to create dynamic tooltip text for a grid.

Example:

$fld = Table_x.Getcell(current_move_line,current_move_col)


·     CURRENT_MOVE_LINE

  
Used with grid (table) object only. Specifies the line over which the cursor moves. This is numeric field, minimum value is 1. Usually used to create dynamic tooltip text for a grid.

Example:

$fld = Table_x.Getcell(current_move_line,1)



·     CURRENT_MOVE_NAME

  
Specifies the name of an object over which the cursor moves. This is alpha field, 30 characters long. Usually used to create dynamic tooltip text.
 


·     CURRENT_OBJECT_NAME

  
Specifies the name of an object which has the focus. This is alpha field, 30 characters long.
 


·     CURRENT_OBJECT_FULLNAME

  
Specifies the qualified name of an object which has the focus. This is alpha field, 120 characters long.

Example:

TAB_1.TABPAGE_1.FIELD_X
 


·     CURRENT_PRINT_POS
  
Current print position, in pixels, numeric field. Precisely, this variable denotes the start of a lineset. Used in reports only. See also END_OF_PAGE.


·     CURRENT_PROCEDURE
  
Name of current procedure/function, alpha field 30 characters long.


·     DEFAULT_PRINTER
  
Name of default printer, alpha variable.
            See Getprintername function.


·     END_OF_PAGE
  
End of page (overflow) position, in pixels. Used in reports only. Combining this value and CURRENT_PRINT_POS, we can force the skip to the new page.

Example:

if current_print_pos + 100 > end_of_page
      skip_before page
end_if
 
  
Note
Run-time engine maintains current print position, break conditions, skip conditions, report layout, etc. These variables (end_of_page, current_print_position should be used in very rare, special circumstances, when you want to change default behavior.

·     EOF
  
End-of-file indicator. Used very rarely with seq files only.

·     IAF_CURRENTLANG
  
Current language in multilingual applications. This is numeric variable, value 1 corresponds to the main (first) language, etc. Multilingual support, languages and language ordering can be defined in 'Multilingual support' dialog.

·     IAF_DISPLAYMODE
  
Numeric variable, value 0 indicates data-entry phase, value 1 indicates display phase. The display phase starts after default reading or after DISPLAY_WINDOW_FIELDS instruction. During this phase, program goes through the subform fields and displays their values. At the end, the variable changes its value to 0 and the first active field after last entered one gets the focus.

·     IAF_PRINTMODE
  
IAF_PRINTMODE is a numeric field. This variable controls print dialog in reports.

Possible values:
  
0 (default)
Print dialog, buttons: Preview, OK, Cancel. 'Preview' creates print file and activates Preview. 'OK' prints to selected printer. 'Cancel' stops the program.
1
No print dialog, output is default printer
2
No print dialog, no print output
3
Print dialog, 'Cancel' disabled
4
Print dialog, buttons: Preview, OK, Cancel. 'Cancel' jumps to ending actions.
5
No print dialog, output is HTML file
6
No print dialog, output is text file
7
Print dialog, 'Preview' disabled
8
Print dialog, 'OK' disabled
9
No print dialog, output is standard Iaf print file
10
No print dialog, output is PDF file. See Create PDF output.
11
Excel output
12
No print dialog, start preview at the end of the program


·     IAF_ROWCOUNT
  
IAF_ROWCOUNT contains the number of rows affected by last EXECSQL statement.

Example

Execsql DELETE FROM tbl WHERE fld = 1
Pause "Number of removed rows is " | iaf_rowcount

·     IAF_SQLERROR
  
IAF_SQLERROR contains five-characters ODBC error code if SQL statement ends with an error. If SQL statement ends successfully, the variable contains SPACE. This variable need to be tested only if WHENEVER SQLERROR CONTINUE is in effect.
Examples of ODBC error codes:
'01000'  General warning
'01004'  String data, right truncated
'08S01' Communication link failure
'22003'  Numeric value out of range
'22007'  Invalid datetime format
'22012'  Division by zero
'HY000' General error
'HY010' Function sequence error
'IM001'  Driver does not support this function
...

·     IAF_SQLNATIVE
  
Numeric variable, integer, contains native error specific to datasource.

·     IAF_TABLEMODE
  
Numeric variable, value 0 indicates data-entry phase, value 1 indicates display phase. Used with data-entry grid (table) only.

·     NUMBER_LABELS
  
Number of output labels. Used in reports only, report type = label. By default, number of output labels is equal to the number of records in input record stream. This value restricts number of output labels. Also, this value defines number of labels if there is no input table.

·     PAGE_COUNTER
  
Current page number in a report.

·     PAGE_OVERFLOW
  
Numeric variable, denotes the reason of jump on the new page. Value 1 indicates overflow (previous page is full), value 0 indicates break. Used in reports only, especially is useful in page footer.

·     REPETITION_LINE
  
Integer numeric variable denoting current repetition, if lineset is repetitive. Initial value is 1.

·     REPORT_ID
  
Integer numeric variable denoting part of the report.
In report header this variable has value -1.
In recapitulation, value is 31.
Otherwise, the value is 1 to 20.

·     STARTING_LABEL
  
Used in reports only, report type = label. By default, starting label is the first label on the page (1).

·     TOTAL_PAGES
  
Total number of pages in a report. This variable can be used in SOURCE statement only, i.e. in 'Source = total_pages'.

·     TOTAL_PAGES_ON_PAGEBREAK
  
Total number of pages between two consecutive page breaks. This variable can be used in SOURCE statement only, i.e. in 'Source = total_pages_on_pagebreak'.

·     TREELEVEL
  
Integer numeric variable denoting the level of selected tree node (1, 2, 3, ...).
If none of the nodes is selected, value is 0.


·     WEBSERV_STATUS
  
Integer numeric variable denoting status of web service call. Value 0 indicates success. Values < 0 indicate URL errors, while values > 0 indicate HTTP errors.
