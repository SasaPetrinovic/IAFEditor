Setnumberlines
   

This method sets number of lines in the grid to be displayed in display phase.

Format

            Setnumberlines(lines)

Parameters


·     lines - number of lines, integer

Applied To

            Data entry grid.

Note

            1/ When we use the grid for updating a table or a file, in 'Table starting actions' we load records using 'Read_all/load_key' statement. The loop 'Read_all ... end_read' fills up internal grid structures and the program knows initial number of the records, record keys, etc. After that, program displays existing lines and moves the cursor to the next row (if new lines enabled, see Enablenewlines), or to the first row if new line cannot be entered. Therefore, we must not use this method in this case. The next two statements are sufficient to initial grid filling:

      read_all/load_key myfile key=...
      end_read

or, if we use database

      read_all/load_key mytbl Where ...
      end_read

            2/ If we use the grid to update arrays, we MUST use this method to specify number of initial rows in the grid. In 'Table starting actions' we fill up arrays and call Setnumberlines:

      $arr1(1) = 10
      $arr1(2) = 20
      $arr1(3) = 30
      $arr2(1) = 'AAA'
      $arr2(2) = 'BBB'
      $arr2(3) = 'CCC'
      table_1.Setnumberlines(3)

            This means the following: initially display three lines and move the cursor to the 4th row.
            If the array is empty, we simply write down

      table_1.Setnumberlines(0)

Conclusion: this method need to be used only if we deal with arrays in data-entry grid. In this case, the method must be invoked in 'Table starting actions'.


Example

  
      table_1.Setnumberlines(counter_1)
