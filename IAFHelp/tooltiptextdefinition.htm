Tooltip text definition
   


The tooltip attribute applies to the following objects:


·     Text label
·     Source field
·     Destination field
·     Combo box
·     Check box
·     Radio button
·     Push button
·     Bitmap button

Define tooltip

Choose tooltip type (static, dynamic, classic, balloon, super, subform). Enter "tooltip text" field, or click "tooltip parameters" button and, in next dialog, enter parameters.

Remove tooltip

Select "Static" radiobutton. "Tooltip text" field will be enabled. Delete this text.



Static tooltip


Static tooltip is defined in the painter and cannot be changed at run-time.

Dynamic tooltip


Dynamic tooltip can be changed at run-time. User should prepare alpha variable containing tooltip text. The "tooltip text" field can be defined using two formats:

            $tooltext

or


            $tooltext = tooltip_func(p1, p2, ...)

where is:
            $tooltext - local alpha variable of arbitrary length,
            tooltip_func - any function (local or global) returning an alpha variable.

In the first format, local variable $tooltext is filled somewhere in the program.
When tooltip text is about to be displayed, run-time engine just picks-up contents of the variable and displays it in tooltip window.

In the second format, run-time engine activates function tooltip_func. This function fills up the output variable, and after that, contents of the variable displays in tooltip window.


Tooltip types

There are four types:


·     classic, plain text displayed on rectangular background.
·     balloon, plain text displayed on rounded rectangle,
·     super, an Office 2007 like tooltip, with three parts: header, body (required) and footer.
·     subform, an ordinary subform plays the tooltip role.


Classic tooltip

            Static

Select "Static" and "Classic" radiobuttons. In "tooltip text" field enter appropriate text, like This is a tooltip text.

More than one line can be displayed in static text . To force line break, insert 'carriage return/line feed' pair using an escape character (\) in the text.

Example

            This is a tooltip text, line 1\r\nLine 2\r\nLine 3

\r denotes carriage return character, \n denotes line feed. At run-time, three lines will be seen on the screen:


This is a tooltip text, line 1
Line 2
Line 3


            Dynamic

Select "Dynamic" and "Classic" radiobuttons. In "tooltip text" field enter variable name

            $tt_text [= tt_function()]

Example:
            function tt_function()
                        $tt_text = "This is a classic tooltip"
            end_function


Balloon tooltip

            Static

Tooltip text is an ordinary text, like This is a tooltip text.

            Dynamic

Tooltip text contains variable name in the form

            $tt_text [= tt_function()]

Example:
            function tt_function()
                        $tt_text = "This is a balloon tooltip"
            end_function

Super tooltip

            Static

To create static super tooltip, select "Static" and "Super" radiobuttons. The button "Tooltip parameters" appears. Click this button, and in the next dialog enter desired text.

            Dynamic

Select "Dynamic" and "Super" radiobuttons. In the "tooltip text" field, enter the name of the variable in the form

            $tt_text [= tt_function()]

Output variable $TT_TEXT should be formatted as follows:

            header_text
            ;                       semicolon, the separator
            1 or 0                (header line, yes/no)
            ;
            body_text
            ;
            body_image or empty string
            ;
            1 or 0                (footer line, yes/no)
            ;
            footer_text
            ;
            gradient_color_begin
            ;
            gradient_color_middle
            ;
            gradient_color_end


Gradient colors are optional.


Example:
            function tt_function()
              $tt_text = compress_sep(';',"Header","1","Body text ...","","0","Footer")
            end_function
            
Subform tooltip

            First, you should paint tooltip subform. In 'Subform properties' dialog, choose 'Subform type = tooltip'. Objects like button, check-box, radio-button cannot be used in tooltip subform.

            Static

To create static subform tooltip, select "Static" and "Subform" radiobuttons. The button "Tooltip parameters" appears. Click the button, and in the next dialog enter subform number and, optionally, select colors.

            Dynamic

Select "Dynamic" and "Subform" radiobuttons. In the "tooltip text" field, enter name of the variable in the form

            $tt_text [= tt_function()]

Output variable $TT_TEXT should be formatted as follows:

            #N                    subform number, required, for example #5
            ;                       semicolon, the separator
            gradient_color_begin
            ;
            gradient_color_middle
            ;
            gradient_color_end

If you want to keep subform color, output variable format is

            #N                    subform number, for example #5


Example:
            function tt_function()
              $tt_text = compress_sep(';','#5','12345','255','255')
            end_function

or


            function tt_function()
              $tt_text = '#5'
            end_function


Note

            CURRENT_MOVE_NAME denotes the object which the tooltip is displaying for. This object can be used in CURENT_MOVE_NAME.Getcell(line,col) or CURENT_MOVE_NAME.Objvalue() methods.

            Internal variables current_move_name, current_move_line and current_move_col can be used to retrieve current cursor position.




Using tooltip in grid

Let's $tooltext is a local variable, 300 characters.


            Example 1

            Tooltip attributes: classic (balloon), dynamic.
            Tooltip text: $tooltext = tt_func1()


            FUNCTION TT_FUNC1() return_value A300
                        tt_func1 = "Lin=" | edit(current_move_line,"ZZZ9") | " Col=" |      edit(current_move_col,"ZZZ9")
            END_FUNCTION

or

            FUNCTION TT_FUNC1() return_value A300
                        define_local $$cell a 10
                        $$cell = current_move_name.Getcell(current_move_line, 1)
                        tt_func1 = "Lin=" | edit(current_move_line,"ZZZ9") | " Col=" |      edit(current_move_col,"ZZZ9")|" col1_value = "|$$cell
            END_FUNCTION


or

            FUNCTION TT_FUNC1() return_value A300
                        define_local $$id a 10
                        $$id = current_move_name.Getcell(current_move_line, 1)
                        select * from tbl where pk = :$$id
                        tt_func1 = "Field = " | tbl.name
            END_FUNCTION



            Example 2

            Tooltip attributes: super, dynamic.
            Tooltip text: $tooltext = tt_func1()

            FUNCTION TT_FUNC1() return_value A300
                        define_local $$id a 10
                        $$id = current_move_name.Getcell(current_move_line, 1)
                        select * from tbl where pk = :$$id
                        tt_func1 = compress_sep(';','Customer','1',tbl.description)
            END_FUNCTION


            Example 3

            Tooltip attributes: subform, static

            Suppose that the tooltip subform is subform 5.
            Somewhere in subform 5, we can read grid cell, for example

            $id = current_move_name.Getcell(current_move_line, 1)

            and using obtained value, read a record

            select * from any_table where prim_key = :$id

            After that, we can display fields from any_table.
