WHENEVER
   

This statement specifies action to be performed when fatal error occurs during the execution of a SQL statement.

Format 1

            WHENEVER SQLERROR STOP_RUN

            Displays an ODBC error message and stops the program. This is the default action.

Format 2

            WHENEVER SQLERROR CONTINUE

            Stores five-characters ODBC error into internal variable IAF_SQLERROR, native error into internal variable IAF_SQLNATIVE and continues. These variables should be tested after each SQL statement in the block.

Scope of the statement is: local procedure, global procedure or any block of statements (subform calculations, field calculations, etc.).

Explanation:


When Iaf uses a database, there exist two types of errors: standard and fatal.
Only three standard errors exist: record not found, record is duplicated and record is locked.
These errors can be processed by Iaf:

            if record REC_XYZ not_found
            if record REC_XYZ duplicated
            if record REC_XYZ locked

All other errors are considered as fatal ones. When they happen, Iaf displays error message, rollbacks the transaction and stops the program.

By using WHENEVER SQLERROR CONTINUE, we can control fatal errors.


Example 1:

Procedure locproc
!! procedure start: whenever sqlerror stop_run is in effect
      whenever sqlerror continue    !! change default behavior
      write tbl1                    !! try to insert record
      if IAF_SQLERROR <> space      !! fatal error
            pause "SQL error: " | IAF_SQLERROR | IAF_SQLNATIVE
            rollback
      else_if record tbl1 duplicated !! standard error
            pause "non-unique key"
            rollback
      else                          !! no error
            commit
      end_if
!! end of procedure
!! return to the standard behavior: whenever sqlerror stop_run
end_procedure


Example 2, referential integrity:

Let's consider two tables defined as follows

Create table tbl1 (
            fld1 char(2) not null primary key,
            fld2 char(10))

Create table tbl2 (
            fld1 char(2) not null primary key,
            fld2 char(2) not null references tbl1 (fld1),
            fld3 int)

If we try to insert a record into tbl2 and related record does not exist in tbl1, we will get an error. The problem is because different databases give different errors:
  
MS SQL Server
IAF_SQLERROR = '23000', IAF_SQLNATIVE = 547
MS Access
IAF_SQLERROR = '23000', IAF_SQLNATIVE = -1613
DB2
IAF_SQLERROR = '23503', IAF_SQLNATIVE = -530
Informix
IAF_SQLERROR = '23000', IAF_SQLNATIVE = -691
Oracle
IAF_SQLERROR = '23000', IAF_SQLNATIVE = 2291
MySQL
IAF_SQLERROR = 'HYT00', IAF_SQLNATIVE = 1452
