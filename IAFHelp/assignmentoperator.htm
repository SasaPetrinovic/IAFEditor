Assignment operator (=)
   

Assigns value(s) to a variable, array or record.


Format 1

            variable = value_expression

            Assigns the value of the right-hand value expression to the left-hand variable.
            The variable on the left side of the equal sign can be:
·                       database field,
·                       local field,
·                       some of internal fields (acc_nn, counter_nn, eof, iaf_printmode, page_counter),
·                       function name (within function body),
·                       argument of the function (within function body),
·                       recap field within define_recap_record block.


Format 2

            array1() = array2()

            Moves all the elements from the righ-hand array to the left-hand array. If the left-hand             array has more elements than the right one, remain elements will be initialized.

            One variation of this format is

            alpha_array() = 'alpha_literal'

            Example:

            If $arr is an array of 10 elements, each element is 1 character long, then

            $arr() = '0000011111'

            has the same effect as

            $arr(1) = '0'
            $arr(2) = '0'
            ...
            $arr(10) = '1'


Format 3

            record_name1 = record_name2

            Moves corresponding fields from the right-hand record buffer to the left-hand record             buffer (see CORR).

Format 4

            filename.FILE_NAME = value_expression

            where filename is a file from Datasource dialog, FILE_NAME is a keyword, and value_expression is alpha literal or alpha variable.
            This format sets physical file name (path).

            Example:

            In Datasource dialog we declare a file as FILE_FROM_DD /MANUAL. This means that the file will not be opened automatically at program start. Later in program we can write

      if file "c:\temp\a.seq" found
            file_from_dd.FILE_NAME = "c:\temp\a.seq"
            open file_from_dd
            ...
            do_something with file_from_dd
            ...
            close file_from_dd
      else
            ...
      end_if


Notes

            Assignment rules for alpha fields


·     alpha_1 = alpha_2   If alpha_2 is longer than alpha_1, result field will be truncated. If alpha_2 is shorter than alpha_1, alpha_2 will be moved to the destination and remain part of alpha_1 will be filled by spaces.

·     partial move in the form alpha_1 = alpha_2[seg_start:seg_length]. If seg_start < 0 or seg_start > length of alpha_2, one question mark (?) will be moved to the alpha_1.


            Assignment rules for numeric fields

            Excess decimals are truncated. If integer part of the source field is greater than integer part of the destination field, 'size error' will be signaled.

            Assignment rules for date/time/datetime fields


·     date_field = datetime_field   Moves date part to the destination field.
·     time_field = datetime_field   Moves time part to the destination field.
·     datetime_field = date_field   Moves date_field into the date part of datetime_field and time part initializes (moves zeros to time part of the datetime_field). To build datetime_field from one date_field and a time_field, use DATETIME function.
