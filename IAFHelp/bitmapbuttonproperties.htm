Bitmapbutton properties
   

Object name

When the new button is created, generator automatically assigns the name Bitmapbutton_nnn, where nnn = 1, 2, …
Object name can be maximum 30 characters long. Allowed characters are A-Z, 0-9, _ and $.
First character must be a letter. Object name is case-insensitive. Object name can be used in Methods.

Member of group (information only, cannot be changed)

If the object is a member of a group, the name of this group is displayed. An object can be the member of one group only.

Caption

The text which appears on the button. If you want to define one letter as a prefix (mnemonic) character, precede it with ampersand (&). At run-time, that letter will be underlined and user can click that button by pressing Alt + letter.
If you want to display single ampersand in the caption, enter two consecutive ampersands.

Within one subform, all prefix characters must be different.

To change caption text dynamically, use the Display method.

Sequence number

If the button has sequence number 0, click is allowed at all times if the button is enabled.

If the button has nonzero sequence, click will be accepted only if all objects with the smaller sequence number are processed.

Visible

Determines whether the button is visible when the subform is first displayed.

Enabled

Determines whether the button is displayed as disabled when the subform is first displayed.
When the button is disabled, mouse clicks are ignored.

Font name

Choose one from offered font names.

Font size

Specifies font size of the text.

Bold

Specify that font weight is 700. Normal weight is 400.

Italic

Specify whether text is italic.

Underline

Specify whether text is underlined.

Tooltip text

Tooltip text is a text that will be displayed when the cursor hovers above the button. See Tooltip text definition.


Button style

Values:

Standard           Standard push button.
Flat                   Flat, not three-dimensional.
Flat with shading

Cancel button

If this attribute is checked, when click occurs, the program immediately leaves current field without further actions. In other words, no field calculations, no field checking, no line ending actions (if we process a grid). All cancel buttons have sequence = 0. See also 'Causes validation' attribute.

Multiline

Word wrap is on. If the caption is too long to fit in a single line, text will be displayed in multiple lines.

Causes validation

See button properties.

Menu

Attaches popup menu to the button. If you select this attribute, context menu for this object contains additional two choices: Define menu layout and Define menu code.

Menu position

Position of the menu in regard to the button, down or right.

Picture

Specifies the path of the picture that will be displayed on the button. The default path is IAFRES, default file extension is ICO. Only ICO or BMP images can be displayed on the button.

The # sign in the first position of the path (for example, #ok.ico) denotes IAFRES folder.

Horizontal text alignment

Specifies the caption (text) position in respect to the picture.

Values:

Down
Up
Right
Left

Text color

Color of text. Double click brings up a color definition dialog.

Active text color

This attribute is applied if button style is Flat. Specifies text color when cursor is above the button.

Active background color

This attribute is applied if button style is Flat. Specifies background color when cursor is above the button.

Active picture

This attribute is applied if button style is Flat. Specifies the picture when cursor hovers above the button. If you do not want to change the picture, leave this field empty.

Shading style

Specifies that Color_from fades gradually to Color_to.

Values:

None                No shading
Horizontal
Vertical

Shading variant

Valid if shading style is Horizontal or Vertical.
Possible values are 1 to 4.

Color from

Starting color. Valid if shading style is Horizontal or Vertical.

Color to

Ending color. Valid if shading style is Horizontal or Vertical.
