CHECK_RECORD
   

This statement controls (non)existing record with given key.

Format


            CHECK_RECORD[/TRIM] file_name [PART_]KEY[n] = key [WHERE cond_exp]


Where


·     file_name - file (table) from where the record is read.
·     n - key number: 1 for primary, .... If omitted, primary key will be used.
·     key - concatenation which represents the key of the file (table).
·     cond_exp - condition for record check.


Qualifiers


            /TRIM trims trailing blanks from last key segment. Record is read with new key length. This applies only if the last key segment is alphanumeric.

            Prefix PART_ can be used only if the number of key segments is greater than 1.


Note

            This statement is mainly intended to use with index-sequential files like Btrieve.

            /TRIM and PART_ can be used with indexed files only. Not available with RDB tables.


Example

  
      CHECK_RECORD file1 KEY="AAA"
      ERRMSG = "Record with key AAA does not exist"

! Another way to control record existence
      READ file1 key="AAA"
      CHECK record file1 found
      ERRMSG = "Record with key AAA does not exist"
 

See also CHECK, READ, SELECT, ERRMSG
