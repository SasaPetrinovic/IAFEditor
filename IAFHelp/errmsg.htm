ERRMSG
   

This statement displays error message if the CHECK expression value is FALSE, and returns
control to the last object having input focus (destination field or push button).

Format 1


            ERRMSG = concatenated_exp

Format 2

            ERRMSG = function(...)


Where


·     concatenated_exp - concatenation of variables, can be:

            - an alpha literal,
            - an alpha variable,
            - STRING function,
            - STRING_EDIT function,
            - any concatenation


·     function - an user function displaying message text

Note

            ERRMSG is always used together with a check (CHECK, CHECK_RECORD) instruction.

            We can use CHECK and ERRMSG (or WARMSG) instructions in:

·     a destination field after DESTINATION instruction to validate entered value,
·     button click actions to validate all subform fields,
·     line_ending actions to validate all the fields in one line of the grid.


Examples

  
      Destination = fld1
      Check fld1 >= 0
      ERRMSG = "Entered number must not be negative"

      !! -or-

      Destination = fld1
      Check fld1 >= 0
      ERRMSG = String(ids_1)
--------------------------------------------------------
      Destination = fld1
      Check fld1 > 0
      ERRMSG = "Value " | fld1 | " is invalid"

      !! -or-

      Destination = fld1
      Check fld1 > 0
      ERRMSG = String_edit(string(ids_2),fld1)
where string IDS_2 contains the text
Value % is invalid

      Destination = fld1
      Check_record file1 KEY=fld1
      ERRMSG = "Record does not exist"

---------------------------------------------------

      check fld1 > 10
