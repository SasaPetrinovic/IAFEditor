Select statement
   

This is a classic SQL statement used to retrieve rows from a RDB table:

            [DISTINCT] select_list FROM table(s)
            [WHERE search_condition]
            [GROUP BY group_list]
            [HAVING search_condition]
            [ORDER BY orderby_list]
            [UNION
            SELECT ...]


            Select_list specifies columns to be returned by the query. The list consists of fields, formulas and functions. If a function or formula has been used, the destination in the form 'INTO :parameter' must be specified.

            Select statement without parameters

            This is the simplest form of the select statement.

            SELECT * FROM tbl
            SELECT fld1,fld2 FROM tbl
            SELECT tbl.* FROM tbl
            SELECT * FROM tbl ORDER BY fld1
            SELECT fld1,fld2 FROM tbl GROUP BY fld1,fld2 ORDER BY fld1,fld2
            SELECT * FROM tbl WHERE fld1 > 100 ORDER BY fld1
            SELECT fld1 AS alias1,fld2 AS alias2 FROM tbl

            Select statement with parameters

            A parameter is a variable which does not belong to the table(s), participating in the select statement. The parameter marker is a colon sign in front of the variable. At run-time, these parameters will be replaced by their values.

            SELECT * FROM tbl WHERE fld1 > :acc_1

            In above statement, :acc_1 is a parameter, because acc_1 is an internal field.

            SELECT * FROM tbl WHERE fld1 BETWEEN :acc_1 AND :acc_2
            SELECT * FROM tbl WHERE fld1 IN (:$var1, :$var2, :$var3)

            As we already said, parameters in select_list are required if the list contains functions or formulas.

            SELECT count(*) INTO :counter_1, SUM(fld1) INTO :acc_1 FROM tbl
            SELECT count(*) INTO :$cnt FROM tbl WHERE fld1 = :$var1
            SELECT fld1,fld2,fld3+fld4 INTO :acc_1 FROM tbl

            Column_alias can be used in ORDER BY, but cannot be used in WHERE,
            GROUP BY and HAVING. Example:

            SELECT fld1,fld2,fld3+fld4 AS formula1 INTO :acc_1 FROM tbl ORDER BY formula1

            Dynamically created select statement

            The parameters cannot be used in each part of the select statement. The following statement is valid

            SELECT * FROM tbl WHERE fld1 > :acc_1 ORDER BY 1
            
but, the next one is invalid

            SELECT * FROM tbl WHERE fld1 > :acc_1 ORDER BY :counter_1

            To solve above problem, we use RT_REPLACE function to create select statement at run-time.

            SELECT * FROM tbl WHERE fld1 > :acc_1 ORDER BY rt_replace($ordvar)

            $ORDVAR is an alpha variable. If it contains 1, created statement at run-time will be

            SELECT * FROM tbl WHERE fld1 > :acc_1 ORDER BY 1

            If $ORDVAR contains 3, created statement at run-time will be

            SELECT * FROM tbl WHERE fld1 > :acc_1 ORDER BY 3

            If we use this form of the select statement, required keywords are SELECT and FROM. All other parts can be replaced by RT_REPLACE.

            SELECT * FROM tbl GROUP BY rt_replace($group)

            If table name should be rt_replaced, all the fields from select_list must be used in the form "field INTO :var".

            SELECT fld1 INTO :$var1 FROM rt_replace($name) WHERE rt_replace($condition)

            where $name, $group, $condition are local alpha variables.


            FROM table(s)

            Specifies table(s) used in the query. The list consists of tables or derived_tables.
            The derived table is a subquery that returns rows from the database.
            All tables from the list, except derived tables, must be declared in Datasource dialog.
            
            The simplest query uses one table only:

            SELECT * FROM tbl

            If we use two or more tables in the list,  these tables are joined. In that case we have to declare relation between the tables.

            SELECT * FROM tbl1, tbl2 WHERE tbl1.f1 = tbl2.f1

            This type of join is inner join. The inner join is the most frequently used join in applications. This join returns only the rows that match both joined tables. The order of the tables in the SQL statement is not important.
            Another way to declare inner join is

            SELECT * FROM tbl1 INNER JOIN tbl2 ON tbl1.f1 = tbl2.f1

            This format is SQL-92 compatible, but the first form is more portable.


            In outer joins, the order of tables in the SQL statement is significant.


            Using table_alias (keyword AS)

            SELECT * FROM tbl1 AS t1, tbl2 AS t2 WHERE t1.f1 = t2.f1

            The keyword AS is optional:

            SELECT * FROM tbl1 t1, tbl2 t2 WHERE t1.f1 = t2.f1

            Using derived table

            SELECT tbl1.*,dertbl.frm FROM tbl1, (SELECT sum(fld) AS FRM from tblx Where tbl1.f1=tblx.f1) AS dertbl

            Self join - join table to itself.

            a/ In Datasource dialog declare two record streams:

            table1, alias=t1
            table1, alias=t2

            SELECT t1.fld1, t2.fld1 FROM table1 AS t1, table1 AS t2 where t1.k1=t2.k1

            Fields in select_list will be stored in appropriate buffers (t1 and t2).

            b/ In Datasource dialog declare only one record stream:

            table1

            and write down a statement like

            SELECT t1.fld1, t1.fld2, t2.fld1 into :$loc FROM table1 t1, table1 t2 where t1.k1=t2.k1

            All the fields from t1 will be stored to buffer 'table1'. Fields from t2 must be stored into parameter variables.

            ORDER BY

            Specifies sort order of the result set.

            SELECT * FROM tbl ORDER BY fld1, fld2
            SELECT * FROM tbl ORDER BY fld1 ASC, fld2 DESC

            Sort key can be any field from the table. However, if SELECT DISTINCT is specified, fields from select-list can be used only. If UNION is specified, the fields from the first select-list can be used.

            SELECT fld1, fld2 FROM tbl1
            UNION
            SELECT fld11, fld12 FROM tbl2
            ORDER BY fld1

            We can use unsigned integers as sort keys. In that case, the integers represent ordinal numbers of the fields in select-list.

            SELECT fld1, fld2, fld3 FROM tbl ORDER BY 1   is the same as
            SELECT fld1, fld2, fld3 FROM tbl ORDER BY fld1
