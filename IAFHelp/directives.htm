Directives (pseudo-commands)
   

Directives are used to change default program behavior. All directives start with number sign (#).

List of directives:

            #DONT_USE_LOCAL_FIELDS
            #DONT_USE_THIS_FUNCTION

            #USE_DECIMAL
            #END_DECIMAL

            #USE_TERMINATOR
            #END_TERMINATOR

            #IGNORE_TABLES
            #END_IGNORE_TABLES

            #ALTER_DECIMAL_POINT
            #RESTORE_DECIMAL_POINT

            #SELECT_BUFFER_SIZE nnnnn


Explanations:

#DONT_USE_LOCAL_FIELDS - local fields starting with one dollar sign ($XXX) cannot be referenced in a function/procedure.

Example

Procedure proc
#dont_use_local_fields
            define_local $$x a 80
            define_local $$a a 20

            $$a = '123'
            $$x = 'abcd'
            if $myfld = 'a'     !!<--- invalid reference
                        ...
            end_if
end_procedure

#DONT_USE_THIS_FUNCTION - a function/procedure which contains this directive cannot be used in a program.

Example

Procedure proc
#dont_use_this_function
            ...
end_procedure


#USE_DECIMAL          - use decimal arithmetic in formulas. Rounding method is defined in Default/Options dialog.
#END_DECIMAL          - stop using decimal arithmetic in calculations.

The generator normally uses standard C-compiler arithmetic in formula calculations. All decimal operands (unpacked, packed) are converted to double-floating values. If all formula operands are integers, then default conversion is into 4-bytes or 8-bytes integers. After calculations take place, the result is converted into appropriate numeric type.

When to use decimal arithmetic? If final result or intermediate result have more than 15 digits, decimal arithmetic should be used. Also, in financial applications, in order to ensure that formula gives absolutely precise value, decimal arithmetic should be used. Precise value is a value calculated by hand.


#USE_TERMINATOR   - use statement terminator, ignore continuation sign. Statement terminator is a semicolon sign (;)
#END_TERMINATOR   - stop using statement terminator.

Example

! Standard SELECT statement with continuation sign
select fld1, fld2 \
            from table1, table 2 \
            where table1.f1 = table2.f1 \
            order by fld1

! SELECT with statement terminator
#use_terminator
select fld1, fld2
            from table1, table 2
            where table1.f1 = table2.f1
            order by fld1;     !!<- statement terminator is semicolon
add 1 to counter_1;
add 2 to counter_2;
#end_terminator
add 1 to acc_1
add 2 ro acc_2
            
#IGNORE_TABLES - do not parse the FROM part of a SQL SELECT statement. All the fields in the select_list have to use INTO :parameter clause.
#END_IGNORE_TABLES - restore default parsing.

Example

#ignore_tables
select   fld1 INTO :$fld1, fld2 INTO :$fld2 \
            from table1, table2 \
            where table1.f1 = table2.f1 order by fld1
#end_ignore_tables


#ALTER_DECIMAL_POINT - decimal point is used in numeric editing and is defined in 'Change setup parameters' dialog.
#RESTORE_DECIMAL_POINT - restore default decimal point.


#SELECT_BUFFER_SIZE nnnnn         Standard buffer size for SELECT statement generation is 16000 bytes. This directive increases buffer size, maximum is 500kb. The directive is valid just for the next SELECT statement.

Example

#select_buffer_size 100000 !! Allocate 100.000 bytes for the next SELECT statement
select * from tbl rt_replace($rtbuf)
