DB_IMPORT
   

This function copies data from an external file into RDB table.

Format

            DB_IMPORT(table_name, input_file,delete_flag,commit_interval,skip,ignoredup,progressbar)

Result Value Type

            Numeric, number of imported records. If schema/table does not exist, result is -1. Result value of -2 indicates error on input file. Result value of -3 indicates invalid input file format. Result value of -4 indicates duplicated key (index) value.

Parameters

·     table_name -name of table to be imported, alpha literal or alpha variable. If you want to import a table into your default schema, supply table name only. If you want to use table from another schema, enter the table name in the form SCHEMA.TABLE_NAME.

·     input_file - input file name, previously created by db_export, alpha literal or alpha variable. The name can be: absolute path, like c:\temp\t1.dbe, or path with logical name, like import_dir:t1.dbe.

·     delete_flag - delete rows flag, integer, optional parameter. If you want to append new records to existing ones, enter zero (0) into this parameter. If you want to delete all the rows before inserting, specify a non-zero (<>0) value. Default is 1.

·     commit_interval - number of records to be committed, integer, optional parameter. If zero, commit will be performed at the end of the file. Default is 0.

·     skip - ignore first N records, integer, optional parameter. Default is 0.

·     ignoredup - ignore unique constraint violation, integer, optional parameter. Enter non-zero if you want to ignore constraint violation and continue record inserting. If you want to stop processing on unique constraint violation, enter zero. Default is 0.

·     progressbar - identifier, name of the progress bar, optional parameter. If specified, this object displays progress of the table import operation.


Note

            To use this function, your program has to be connected to a datasource. To enable datasource connection, specify at least one RDB table in Datasource dialog. Only tables from the current connection can be imported. If the program uses multiple connections, the first connection will be used for import.


Examples

  
      $status = db_import("mytbl","c:\temp\mytbl.dbe")
      $status = db_import("mytbl","expdir:mytbl.dbe")
      $status = db_import("myschema.mytbl","expdir:mytbl.dbe")

      $status = db_import("mytbl","expdir:mytbl.dbe",1,1000)
      $status = db_import("mytbl","expdir:mytbl.dbe",,,,,progress_1)
