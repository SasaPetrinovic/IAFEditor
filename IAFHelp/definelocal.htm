DEFINE_LOCAL
   

This statement defines local field in a procedure or function.

Format 1 for alphanumeric, date and time fields

            DEFINE_LOCAL field_name  type size  [num_el] [INIT_ALWAYS[=value]]

Format 2 for numeric fields

            DEFINE_LOCAL field_name  type  num_dig[,num_dec]  [data_type]  [num_el] [INIT_ALWAYS[=value]]

Format 3 for datetime and string fields

            DEFINE_LOCAL field_name  type [num_el] [INIT_ALWAYS[=value]]

Where


·     field_name - a name of the local field (the name has to start with two dollar signs, $$)
·     type - type of the local field (A - alphanumeric, N - numeric, D - date, T - time, DT - datetime, W - Unicode UTF-16, S - ANSI string, SW - UTF-16 string).
·     size - size of the local field. If the field is alphanumeric, field size is between 1 and 16000. If the field is date, size has to be 6 or 8, while time field can have size 4 or 6.
·     num_dig - total number of digits of numeric field.
·     num_dec - number of decimals of numeric field (must not be greater than the number of digits)
·     data_type - numeric field data type. Possible values: U (unpacked) is default, P - BCD packed, W - integer word (2 bytes), L - integer longword (4 bytes), Q - integer quadword (8 bytes), F - floating (4 bytes), D - double floating (8 bytes).
·     num_el - number of elements if the local field is array
·     INIT_ALWAYS[=value] - initialize local variable every time when procedure/function is invoked. If value is omitted, the field will be initialized according to its type. For arrays, value must be omitted.

Notes

            Local fields, defined by this statement cannot be seen outside the procedure/function.

            When the procedure / function is invoked for the first time, these fields have initial values. Initial value for alpha field is SPACE, for numeric is ZERO, for date is ZERO_DATE, etc. After that, the fields keep their previous values. If the field have to be initialized every time when procedure/function is invoked, use INIT_ALWAYS. According C language notation, such fields are static.

            Strings (type = S or SW) are always initialized at the start of procedure/function.

Examples

  
      Procedure myproc
      define_local $$loca a 5       !! alpha field, size = 5
      define_local $$locarr a 5 80  !! alpha array, 80 elements
      define_local $$loc1 n 5
! $$loc2 is numeric field:
!     integer part = 9 digits,
!     number of decimals = 2,
!     total digits = 11
      define_local $$loc2 n 11,2
      define_local $$loc3 d 8
      define_local $$dt dt
      define_local $$nump n 11,2 p  !! BCD packed numeric
      define_local $$numarr n 11,2 p 100  !! numeric array
      define_local $$s s            !! ANSI string
      define_local $$sw sw          !! UTF-16 string

      define_local $$a2 a 3 init_always
      define_local $$a3 a 3 init_always='abc'
      define_local $$a4 a 1 init_always=chr(9)
      define_local $$n1 n 11,2 init_always=12,34
      define_local $$n2 n 6 init_always=counter_1
      define_local $$d1 d 8 init_always=today
      define_local $$d2 d 8 init_always=add_days(today,15)
      define_local $$ts dt init_always=timestamp

      $$loc1 = acc_1
      $$loc2 = $$loc1 * 100
      $$loc3 = add_days(today,15)
      $$numarr(5) = 123,50
      end_procedure
