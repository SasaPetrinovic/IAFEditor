Symbols
   

Symbols represent one of mechanisms of inter-program communication. One program writes a symbol using WRITE_EXTERNAL statement. Another program reads that symbol by READ_EXTERNAL statement.

Internally, symbols can be stored in the two ways.

Symbols in internal file symntt00.iaf

If we use ordinary symbol names, like

write_external/NAME: symnam1 ...
read_external/NAME: symnam1 ...

symbols will be stored in internal file symntt00.iaf. This is standard text file which resides in the same directory as executable program. Each symbol definition occupies one line in the file. Line structure is the following:

symbol_name:=="symbol_value"

If Write_external stores many fields simultaneously, the structure is

symbol_name:=="field1_value<TAB>field2_value<TAB>...fieldN_value"

Symbol name is maximum 30 characters long, symbol_value length is up to 250 bytes.


Symbols in System Registry (for Windows operating systems only)

If we use symbol names starting with dollar sign, like

write_external/NAME: $symnam1 ...
read_external/NAME: $symnam1 ...

symbols will be stored in System Registry under the key HKEY_CURRENT_USER\Software\Sing\Iafw\RT.

The statement write_external/name: $sym1 "1" stores symbol hkey_current_user\software\sing\iafw\rt\sym1, value = "1".

Also, we can read/write any symbol under HKEY_CURRENT_USER branch. The format is

write_external/NAME: $hkey_current_user\sub_key1\...\syb_keyN\symbol_name ...
read_external/NAME: $hkey_current_user\sub_key1\...\syb_keyN\symbol_name ...

If symbol name contains spaces, quotation marks have to be used as delimiters:

write_external/NAME: "$hkey_current_user\sub key with spaces\symbol_name" ...

See Read_external, Write_external
