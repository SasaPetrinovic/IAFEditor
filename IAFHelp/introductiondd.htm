Introduction
   

Data Dictionary utility enables you to define DD entities: global fields, records, files, strings and applications. Also, RDB table definitions can be loaded from an ODBC datasource into the DD.

A global field is a template which defines data type (alpha, numeric, date, time), size, default edit-mask and other attributes. We have to define global fields prior to move to Record definition. When the new DD has been created, several standard global field definitions are stored in it.

Record is a structured set of fields. Besides global attributes, each field can have local attributes valid in that definition only: column header, help text, default value, range control and the description.

File is collection of records containing data. Usually, one file has one record structure. Sometimes, we need more that one record structure in a file. Up to 8 (eight) redefinitions are supported.

RDB table is a table stored in a database, like MS SQL Server, Oracle, DB2, MySQL, etc. Prior to use tables in our programs, we have to load their definitions into the DD.

String is any text displayed on the screen in a message box, as text label, etc.

Application is set of programs and data processed by those programs.




·     If our application processes database tables only, we do not need to create records and files. Just load RDB tables into the DD, and go to a generator.

·     But, if we want to exchange data with another applications via files, or if we want to save some data in work files, we have to create record and file definitions.
