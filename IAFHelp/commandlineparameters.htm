Command line parameters
   

Programs generated by Iaf can use command line parameters. There are three named parameters recognized by an Iaf program: WORK, DSN and PARAM.

WORK - specifies current (working) directory for the program. Parameter value must contain fully qualified path. Quotation marks are required if the path contains spaces. If this parameter does not appear in the command line, the program uses a previously defined working directory.

Current (working) directory is the location on which will be created internal files during the sort.

Examples:

command_line> program.exe WORK=C:\MYPROG
command_line> program.exe WORK="c:\Program Files\myprog"


DSN - datasource name. Program connects to datasource defined by this parameter. If this parameter does not exist, program checks existence of the DSN_applname symbol  in the Registry (key value is HKEYCurrentUser\Software\Sing\Iafw\RT). If the symbol does not exist, program uses datasource defined by the first table in the datasource dialog. Let's remember that 'Load data definition' records table definition and appropriate datasource name into Iaf DD.

Example:

command_line> program.exe DSN=GK;UID=myuser;PWD=mypwd


PARAM - value of this command line parameter Iaf stores into the internal variable CMD_PARAM. CMD_PARAM is an alpha variable, maximum length 255 characters. If the parameter value contains spaces, quotation marks must be used.

Examples:

command_line> program.exe PARAM=p1
command_line> program.exe PARAM="p1"

At the very beginning, program stores value p1 into the CMD_PARAM. Later in the program, we can use CMD_PARAM in an assignment:

            $param = cmd_param   !! $param contains "p1"


Many values can be stored in PARAM:

command_line> program.exe PARAM="p1;p2;p3;p4"

In order to obtain particular value, we use the SPLIT function:

            $arr() = SPLIT(';', cmd_param)
