Editor rules
   

The maximal number of characters in a line is 128.
Each line contains just the one instruction. There is no a termination character. The last non-blank character in the line denotes the end of the instruction.

Examples:

counter_1 = 10
acc_1 = 100,00
add 10 to acc_1
read_external $loc


If an instruction is longer than 128 characters, the continuation sign \ (backslash) must be used:

select fld1, fld2, fld3, fld4 from table1, table2 \
      where table1.fld1 = table2.fld1 and fld2 < 1000 \
      order by fld1, fld2

Also, there is a directive (#use_terminator) which enables the use of statement terminator, see Directives.


Instructions are case-insensitive. The next two instructions are the same:

acc_1 = 100,00
ACC_1 = 100,00

The only exceptions are alpha literals. The next two instructions gives different results:

$x20 = "abcdef"
$x20 = "ABCDEF"


If the first non-blank character in the line is ! (exclamation sign), that line is considered as a comment line.

read_external/name:mysymb $loc
! This is the comment line
! This is the next comment line
$loc = $loc + 2

To write comment in the same line, we must use two exclamation signs:

read_external/name:mysymb $loc !! reading of symbol


If we want to directly incorporate C++ code in the program, we use !C++
in the beginning of the line:

!C++ int result = strncmp(x1, x2, 10);
